# Базы данных, которые можно хостить в Coolify

Coolify поддерживает множество баз данных для различных нужд приложений. Даже если вы не видите здесь свою любимую БД, вы всё равно можете запустить её в Coolify, используя Docker.

Coolify предлагает установку «в один клик» для следующих баз данных:

- [PostgreSQL](/databases/postgresql)
- [Redis](/databases/redis)
- [DragonFly](/databases/dragonfly)
- [KeyDB](/databases/keydb)
- [Clickhouse](/databases/clickhouse)
- [MongoDB](/databases/mongodb)
- [MySQL](/databases/mysql)
- [MariaDB](/databases/mariadb)

Каждая база данных имеет свои сильные стороны и сценарии использования. Нажмите на название БД, чтобы узнать больше.

# Развертывание базы данных

При выборе «New Resource» (Новый ресурс) вы можете выбрать базу данных из списка.

![New Resource](https://coolify.io/docs/images/screenshots/How-to-add-a-database.webp)

Вы можете настроить базу данных простым кликом. Coolify поддерживает следующие БД:

- PostgreSQL
- MySQL
- MariaDB
- MongoDB
- Redis
- DragonFly
- KeyDB
- Clickhouse

## Маппинг портов vs Публичный порт

### Маппинг портов (Ports Mapping)

Маппинг портов использует функцию [проброса портов](https:/.docker.com/network/#published-ports) Docker. Она используется для сопоставления порта контейнера с портом хоста. Например, если вы установите маппинг `8080:80`, порт `80` внутри контейнера будет сопоставлен с портом `8080` хост-машины.

### Публичный порт (Public Port)

Публичный порт используется для открытия порта контейнера в интернет путем запуска TCP-прокси Nginx.

### Что мне использовать?

- Маппинг портов делает соединение/порт постоянным (вам нужно перезапустить базу данных, чтобы изменить его).
- Публичный порт делает соединение/порт динамическим (вы можете изменить его без перезагрузки базы данных, Coolify перезапустит TCP-прокси Nginx за вас).

## Доступ к базе данных во время сборки

Если вы используете билдпак `Nixpacks`, есть два способа получить доступ к базе данных во время сборки:

1. База данных и ваше приложение находятся `в одной сети`: вы можете подключиться к ней, используя `internal URL` (внутренний адрес), предоставленный Coolify.
2. База данных и ваше приложение `находятся в разных сетях`: вам нужно настроить базу данных так, чтобы она была `Accessible over the internet` (доступна через интернет), и использовать `public URL` (публичный адрес), предоставленный Coolify.
