# SSL для базы данных <Badge type="warning" text="^v4.0.0-beta.399" />

Функция SSL для баз данных в Coolify шифрует обмен данными между вашими приложениями и базами данных, гарантируя безопасность данных.

Благодаря автоматической привязке и генерации сертификатов, эта функция упрощает настройку защиты. Она была впервые представлена в Coolify версии **v4.0.0-beta.399**.


## Введение
SSL для базы данных добавляет дополнительный уровень безопасности путем шифрования данных, которыми вы обмениваетесь с вашей БД. Это руководство охватывает:
- Включение режима SSL для подключений к базе данных.
- Выбор подходящего режима SSL в зависимости от ваших потребностей в безопасности.
- Управление сертификатом CA, который проверяет подключения к базе данных.


## 1. Как включить SSL для базы данных
Чтобы защитить соединение с базой данных с помощью SSL:

1. **Зайдите в настройки базы данных**  
   В дашборде Coolify перейдите в общие настройки (General Settings) базы данных, которую вы хотите защитить.
   <ZoomableImage src="/docs/images/database/ssl/1.webp" alt="скриншот дашборда Coolify" />

2. **Включите режим SSL**  
   Отметьте опцию **Enable SSL**, чтобы активировать SSL для соединения с базой данных.
   <ZoomableImage src="/docs/images/database/ssl/2.webp" alt="скриншот дашборда Coolify" />

3. **Выберите режим SSL**  
   Выберите режим SSL из выпадающего меню. Например, выберите **verify-full** для максимальной безопасности.  
   <ZoomableImage src="/docs/images/database/ssl/3.webp" alt="скриншот дашборда Coolify" />

::: warning Примечание
Чтобы использовать SSL после его включения, вам нужно использовать новый URL соединения (connection URL) для вашего приложения, который включает конфигурацию SSL.

Если вы не используете новый URL, соединение с базой данных (в большинстве случаев) не будет использовать SSL.
:::

Coolify автоматически привязывает сгенерированные сертификаты и ключи к необходимым местам, поэтому ручные изменения требуются только в том случае, если вы хотите использовать собственные сертификаты.

## 2. Описание режимов SSL
Coolify поддерживает несколько режимов SSL, каждый из которых обеспечивает различный уровень безопасности:
<ZoomableImage src="/docs/images/database/ssl/4.webp" alt="скриншот дашборда Coolify" />

### PostgreSQL
- **allow (insecure)**  
  Этот режим разрешает как зашифрованные, так и незашифрованные соединения. Он не принуждает к использованию SSL, поэтому в случае сбоя SSL соединение откатится до незашифрованного состояния.
  
  Этот вариант считается небезопасным, так как допускает незашифрованный трафик.

- **prefer (secure)**  
  В этом режиме Coolify сначала попытается использовать SSL. Если SSL-соединение доступно, оно будет использовано, в противном случае будет выполнен откат к незашифрованному соединению.
  
  Хотя этот вариант отдает приоритет шифрованию, он не гарантирует, что каждое соединение будет защищено.

- **require (secure)**  
  Этот режим требует, чтобы соединение было обязательно зашифровано. Однако он не выполняет никаких проверок сертификата сервера.
  
  Это означает, что соединение зашифровано, но подлинность сервера не проверяется.

- **verify-ca (secure)**  
  Этот вариант идет на шаг дальше: он шифрует соединение и проверяет, что сертификат сервера подписан доверенным центром сертификации (CA). При этом не проверяется соответствие имени хоста (hostname) сертификату.
  
  Этот режим предлагает баланс между безопасностью и простотой настройки.

- **verify-full (secure)**  
  Это самый безопасный режим. Он не только шифрует соединение и проверяет центр сертификации, но и подтверждает, что имя хоста сервера совпадает с указанным в сертификате.
  
  Это дает полную уверенность в том, что вы подключаетесь к нужному серверу (аналогично безопасности настроек Cloudflare Origin Certificate).

### Другие базы данных
- **MySQL и MongoDB:**  
  Доступны только следующие режимы: **prefer, require, verify ca, verify full**.
- **MariaDB, Redis, KeyDB, DragonFly DB:**  
  Режимы SSL не отображаются в интерфейсе.
- **Clickhouse DB:**  
  SSL не поддерживается; нет возможности включить его или выбрать режим.

::: warning Заметка для разработчика
Режимы ниже **require** не являются на 100% безопасными, так как они только шифруют соединение без полной проверки подлинности сервера.

Для режимов выше **require** (т.е. **verify-ca** и **verify-full**) вы должны примонтировать CA-сертификат Coolify в контейнер, который подключается к базе данных, для дополнительной безопасности.

Обратите внимание, что в большинстве случаев (например, для PostgreSQL) простое включение SSL ничего не дает, пока вы не начнете использовать новый URL подключения, принудительно использующий SSL.

Тем не менее, для некоторых баз данных (например, на базе Redis) включение SSL в интерфейсе сразу активирует выбранный режим.
:::

::: success Совет
Для максимальной безопасности рекомендуется режим **verify-full** (когда он доступен).
:::


## 3. Управление сертификатом CA SSL
Coolify автоматически управляет сертификатом CA, обеспечивая валидацию защищенных соединений с базой данных.

В дашборде, в разделе **Servers > ИМЯ_ВАШЕГО_СЕРВЕРА > Proxy > Advanced**, вы можете увидеть следующие опции:
<ZoomableImage src="/docs/images/database/ssl/5.webp" alt="скриншот дашборда Coolify" />

- **CA SSL Certificate**: отображает текущий используемый сертификат CA.
- **Save Certificate**: позволяет сохранить локальную копию сертификата.
- **Regenerate Certificate**: позволяет сгенерировать новый сертификат CA при необходимости.


### Рекомендуемая конфигурация
Для безопасных соединений примонтируйте CA-сертификат Coolify во все контейнеры, которым необходимо подключаться к вашим базам данных.

Рекомендуемый маунт (bind mount):

```sh
/data/coolify/ssl/coolify-ca.crt:/etc/ssl/certs/coolify-ca.crt:ro
```


## 4. Использование собственного сертификата CA
Если вы хотите использовать собственный сертификат CA вместо сгенерированного Coolify:

1. **Подготовьте сертификат CA**  
   Убедитесь, что ваш сертификат находится в формате PEM.

2. **Загрузите сертификат**  
   Загрузите ваш кастомный сертификат CA по следующему пути:
   ```sh
   /data/coolify/ssl/coolify-ca.crt
   ```

3. **Примонтируйте сертификат**  
   Убедитесь, что контейнер, которому требуется доступ к базе данных, монтирует сертификат по пути:
   ```sh
   /data/coolify/ssl/coolify-ca.crt:/etc/ssl/certs/coolify-ca.crt:ro
   ```
