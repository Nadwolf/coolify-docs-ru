# PostgreSQL

![PostgreSQL](https://coolify.io/docs/images/database-logos/postgresql.webp)

## Что такое PostgreSQL

PostgreSQL — это продвинутая объектно-реляционная система управления базами данных с открытым исходным кодом, известная своей надежностью, богатством функций и производительностью. Она разрабатывается более 30 лет и широко используется в индустрии.

PostgreSQL (часто просто «Postgres») использует и расширяет язык SQL в сочетании со множеством функций, которые позволяют безопасно хранить и масштабировать самые сложные рабочие нагрузки данных.

## Импорт резервных копий (Import Backups)

Coolify может импортировать дамп базы данных в работающий экземпляр PostgreSQL, используя раздел
**Import Backups** в конфигурации экземпляра.

Дамп базы данных может быть либо файлом, загруженным на сервер, либо файлом, перетащенным в область загрузки прямо на экране конфигурации.

Команду импорта можно настроить, но по умолчанию она ожидает дамп базы данных,
созданный с помощью команды `pg_dump` с флагом `-Fc` (кастомный формат).

Например, следующая команда подключается к базе данных PostgreSQL, работающей в
локальном Docker-контейнере с именем `pg-db`, под пользователем `postgres` и
записывает дамп базы данных `postgres` в файл `example-database.sql.gz`:

```bash
docker exec pg-db pg_dump -U postgres -d postgres -Fc >example-database.sql.gz
```

### Примечание по обновлению PostgreSQL

**Кастомный** формат дампа (`custom format`) чувствителен к различиям версий между командами создания дампа и его восстановления.

Используйте обычный (`plain`) или **tar** форматы дампа для миграции со старой версии
PostgreSQL на новую. При использовании дампов в обычном формате (`plain`) используйте `psql` в качестве команды импорта вместо `pg_restore`.

Дополнительную информацию см. в документации PostgreSQL по командам `pg_dump` и `pg_restore`.
