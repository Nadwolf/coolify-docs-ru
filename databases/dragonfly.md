# DragonFly

![dragonfly](https://coolify.io/docs/images/database-logos/dragonfly-dark.svg)

## Что такое DragonFly

DragonFly — это современное хранилище данных в оперативной памяти (in-memory datastore), созданное как альтернатива Redis с улучшенной масштабируемостью и эффективным использованием ресурсов. Оно предлагает API, совместимый с Redis, обеспечивая при этом более высокую производительность на многоядерных системах. DragonFly разработано для обработки сценариев с высокой пропускной способностью и больших наборов данных более эффективно, чем традиционные хранилища.

Благодаря своей многопоточной архитектуре и продвинутым структурам данных, DragonFly стремится обеспечить улучшенную масштабируемость и производительность для приложений, которым требуется функциональность, подобная Redis, на современном оборудовании.

## Персистентность данных (Data Persistence)

По умолчанию Dragonfly DB не сохраняет данные на диск. Чтобы включить персистентность, настройте создание снимков (snapshots) вручную.

Например, настройте сервис следующим образом:

```yaml
services:
  dragonfly:
    command: 'dragonfly --requirepass XXXXXXXX --dir /data --dbfilename dragonfly-snapshot-{timestamp} --snapshot_cron "*/5 * * * *"'
```

Вы также можете инициировать ручное сохранение с помощью команды `SAVE` через SDK.
