# Руководство по участию в создании документации Coolify

В этом руководстве описан процесс внесения обновлений и исправлений в нашу документацию. Пожалуйста, следуйте этим шагам для обеспечения плавного и эффективного рабочего процесса.


## 1. Работа с репозиторием
- **Процесс релизов:**  
  Мы придерживаемся еженедельного цикла производственных релизов. Ветка **main** представляет собой продакшн, а ветка **next** используется как основная ветка разработки.

- **Правила работы с ветками:**  
  - **Не создавайте Pull Request (PR) в ветку main.**  
  - Все вклады должны направляться в ветку **next**.
  - **Клонируйте репозиторий из ветки next** в свой аккаунт GitHub, а затем начинайте работу над изменениями.


## 2. С чего начать

### Шаг 1: Форк и клонирование репозитория
- Сначала сделайте форк репозитория документации в свой аккаунт GitHub, затем клонируйте свой форк на локальную машину:
  ```sh
  git clone https://github.com/vash-username/nazvanie-repo.git
  ```
- Перейдите в папку склонированного репозитория:
  ```sh
  cd nazvanie-repo
  ```

### Шаг 2: Установка зависимостей и запуск сервера разработки
Для локальной разработки мы рекомендуем использовать [bun](https://bun.sh/). Если вы решите использовать другой менеджер пакетов, пожалуйста, **не включайте его конфигурационные файлы** в свой коммит.

Для настройки среды выполните:
```bash
bun install && bun run dev
```

Сервер разработки запустится на `localhost` через порт `5173`. Вы можете просмотреть документацию по адресу:
```bash
http://localhost:5173/docs/
```


## 3. Требования к изображениям
- **Формат:**  
  Все изображения в документации должны быть в формате `.webp`.
  
- **Расположение:**  
  Помещайте все файлы изображений в директорию `/docs/public`.

- **Использование:**  
  Используйте компонент ZoomableImage для вставки изображений:
  ```vue
  <ZoomableImage src="put-k-izobrazheniyu.webp" alt="Описание изображения" />
  ```


## 4. Написание и структурирование контента

### Рекомендации по документации:
- **Ясный и простой язык:**  
  Используйте простой и понятный язык. Помните, что не все читатели являются носителями английского языка (или русского, в зависимости от раздела).
  
- **Дружелюбность к новичкам:**  
  Разбивайте инструкции на небольшие, легкие для выполнения шаги. Добавляйте скриншоты везде, где это возможно, чтобы визуализировать процесс, особенно для пользователей, которые впервые сталкиваются с self-hosting или Coolify.

- **Организация контента:**  
  Структурируйте контент с помощью четких заголовков, маркированных списков и нумерованных шагов. Это облегчает восприятие информации.


## 5. Отправка вашего вклада
1. **Закоммитьте изменения в свой репозиторий.**  

2. **Создайте Pull Request:**  
   - Откройте PR для слияния ваших изменений в ветку **next** основного репозитория документации. 
   - Предоставьте подробное описание ваших обновлений, чтобы помочь мейнтейнерам эффективно проверить ваш вклад.


## 6. Вопросы и поддержка
Если у вас возникли вопросы или проблемы в процессе:
  - **Создайте Issue:** Откройте тикет в репозитории с описанием проблемы.
  - **Discord:** Свяжитесь с нами в канале contribute в [сообществе Coolify в Discord](https://coolify.io/discord).


## 7. Важные примечания
- **Обновление документации:**  
  Текущая документация может быть немного устаревшей и в ней может не хватать некоторой информации. Мейнтейнеры активно переписывают части документации для улучшения структуры и ясности.

- **Одобрение PR:**  
  Слияние вашего PR зависит от результатов проверки мейнтейнерами. Пожалуйста, наберитесь терпения.


Мы ценим ваш вклад и усилия по улучшению документации Coolify для всех!
