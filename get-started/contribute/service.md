# Добавление нового шаблона сервиса в Coolify

Сервисы в Coolify — это шаблоны, созданные на основе обычных файлов [docker-compose](https://docs.docker.com/reference/compose-file/) с добавлением некоторой «магии» Coolify.

::: info
Ознакомьтесь с [спецификациями docker-compose Coolify](/knowledge-base/docker/compose#coolify-s-magic-environment-variables), чтобы узнать больше о магии Coolify и о том, как извлечь выгоду из сгенерированных переменных и управления хранилищем. Пожалуйста, используйте эту магию при создании PR, чтобы упростить процесс слияния.
:::

1. Добавьте метаданные

   В верхней части вашего файла `docker-compose` добавьте следующие метаданные:

   ```yaml
   # documentation: https://docs.example.com/
   # slogan: Краткое описание вашего сервиса.
   # tags: tag1,tag2,tag3
   # logo: svgs/your-service.svg
   # port: 1234
   ```

   - `documentation`: Ссылка на официальную документацию сервиса.
   - `slogan`: Короткое описание сервиса.
   - `tags`: Список тегов через запятую для лучшего поиска.
   - `logo`: Путь к логотипу сервиса (см. шаг 3).
   - `port`: Основной порт входа в сервис.

::: warning Внимание
Всегда указывайте порт, так как Caddy Proxy не может автоматически определить порт сервиса.
:::

2. Создайте файл docker-compose

   Ниже метаданных добавьте конфигурацию docker-compose. Используйте магию переменных окружения Coolify [здесь](/knowledge-base/docker/compose#coolifys-magic-environment-variables).

   Пример:

   ```yaml
   services:
     app:
       image: your-service-image:tag
       environment:
         - DATABASE_URL=${COOLIFY_DATABASE_URL}
       volumes:
         - ${COOLIFY_VOLUME_APP}:/data
   ```

   **Использование обязательных переменных окружения:**
   При создании шаблонов сервисов помечайте критически важные настройки как обязательные для удобства пользователей:

   ```yaml
   services:
     app:
       image: your-service:latest
       environment:
         # Обязательно — критическая настройка, которую пользователь должен задать
         - DATABASE_URL=${DATABASE_URL:?}
         - API_KEY=${API_KEY:?}

         # Обязательно с разумным значением по умолчанию — улучшает удобство
         - PORT=${PORT:?8080}
         - LOG_LEVEL=${LOG_LEVEL:?info}

         # Опционально — параметры, которые можно оставить пустыми
         - DEBUG=${DEBUG:-false}
         - CACHE_TTL=${CACHE_TTL:-3600}
   ```

   Это помогает пользователям понять, какие настройки необходимы, и предотвращает сбои при развертывании.

3. Добавьте логотип

   - Создайте или найдите логотип в формате SVG (настоятельно рекомендуемый формат).
   - Если SVG недоступен, используйте высококачественный .webp или JPG в крайнем случае.
   - Добавьте файл логотипа в папку `svgs` в репозитории Coolify.
   - Имя файла логотипа должно точно совпадать с именем сервиса в docker-compose.
     - Например, если имя сервиса `wordpress`, логотип должен называться `wordpress.svg`, а итоговый путь — `svgs/wordpress.svg`. Используйте этот путь в метаданных `logo`.

4. Протестируйте ваш шаблон

   Используйте вариант развертывания `Docker Compose Empty` в Coolify, чтобы проверить свой шаблон. Этот процесс имитирует развертывание сервиса в один клик.

5. Создайте Pull Request

   Как только ваш шаблон заработает корректно:

   - Откройте [PR](https://github.com/coollabsio/coolify/compare).
   - Добавьте свой новый файл `<service>.yaml` в папку `/templates/compose`.
   - Включите файл логотипа в папку `svgs`.

   ::: info
   Coolify использует [спарсенную версию](https://github.com/coollabsio/coolify/blob/main/templates/service-templates.json) шаблонов для развертывания.
   :::

## Добавление документации для нового сервиса

После того как ваш шаблон сервиса будет принят в Coolify, важно также добавить документацию для него.
В разделе [Участие в написании документации](/get-started/contribute/documentation) мы объясняем, как запустить документацию на своем ПК.
Как только все будет готово, следуйте этим шагам:

1. Добавьте логотип сервиса в `/docs/public/images/services/`.

2. Создайте файл документации

   Создайте `/docs/services/<service-name>.md` с фронтметтером:

   ```yaml
   ---
   title: "Название сервиса"
   description: "Здесь вы найдете документацию по хостингу [Название сервиса] с помощью Coolify."
   ---
   ```

3. Напишите документацию

   Начните писать документацию после фронтметтера. Используйте следующий шаблон:

   ```markdown
   # [Название сервиса]

   <ZoomableImage src="/docs/images/services/service.svg" alt="Дашборд сервиса" />

   ## Что такое [Название сервиса]?

   Краткое описание и варианты использования.

   ## Ссылки

   - [Официальный сайт](https://example.com?utm_source=coolify.io)
   - [GitHub](https://github.com/example/repo?utm_source=coolify.io)
   ```

4. Добавьте сервис в обзор сервисов (Services Overview)

   - Добавьте новый сервис в список в файле `docs\.vitepress\theme\components\Services\List.vue`, соблюдая формат:

   ```js
    {
        name: 'Название сервиса',
        slug: 'service-name', // Должно совпадать с именем файла документации
        icon: '/docs/images/services/service.svg', // Путь к логотипу
        description: 'Краткое описание сервиса.',
        category: 'Analytics' // Выберите подходящую категорию
    },
   ```

5. Создайте Pull Request

   - Целевая ветка — `next`.
   - Проверьте корректность отображения документации с помощью `bun run dev`.

# Запрос нового сервиса

Если есть шаблон сервиса, который вы хотели бы видеть в Coolify:

1. Поищите в [обсуждениях GitHub](https://github.com/coollabsio/coolify/discussions/categories/service-template-requests) существующие запросы.
2. Если сервис уже запрашивали, проголосуйте за него. Если нет — создайте новый запрос.
