# Авторизация

Для запросов к API требуется токен `Bearer` в заголовке `Authorization`, который можно сгенерировать в пользовательском интерфейсе.

## Доступ

Доступ к API можно получить по адресу `http://<ip>:8000/api`.

За исключением маршрутов `/health` и `/feedback`, все остальные пути имеют префикс `/v1`, что дает базовый URL `http://<ip>:8000/api/v1`.

## Генерация токена

1. Перейдите в раздел **Keys & Tokens** / **API tokens**.
2. Укажите имя для вашего токена и нажмите **Create New Token**.

::: success Совет
Вы увидите токен только один раз, поэтому обязательно скопируйте его и сохраните в надежном месте.
:::

## Область видимости (Scope)

Токен сможет получить доступ только к тем ресурсам, которые принадлежат команде, для которой этот токен был создан.

```php
# Пример токена
3|WaobqX9tJQshKPuQFHsyApxuOOggg4wOfvGc9xa233c376d7
```

## Права доступа (Permissions)

::: warning ВНИМАНИЕ!
Часть данных API не будет возвращена, если у токена API нет соответствующих прав доступа.
:::

На данный момент существует три типа прав доступа:

- `read-only` (по умолчанию)
- `read:sensitive`
- `view:sensitive`
- `*` (все права доступа)

### `read-only`

С этим разрешением вы можете только читать данные из API, но не можете создавать, обновлять или удалять какие-либо ресурсы. Также вы не сможете видеть конфиденциальные (sensitive) данные.

### `read:sensitive`

С этим разрешением вы можете только читать данные из API и видеть конфиденциальную информацию, которая обычно скрыта. Вы не можете создавать, обновлять или удалять ресурсы.

### `view:sensitive`

Без этого разрешения пароли, API-ключи и другие чувствительные данные будут скрыты (заменены на `redacted`) в ответах API.

### `*`

Полный доступ ко всем ресурсам и конфиденциальным данным.
