# Нестабильное соединение с сервером

Если вы сталкиваетесь с частыми обрывами соединения, в 90% случаев причина кроется в настройках брандмауэра (firewall), UFW, iptables или в сетевых конфигурациях между вашим экземпляром Coolify (или Coolify Cloud) и целевым сервером.

## Симптомы

- Сервер то доступен, то нет.
- Вы получаете много писем об обрыве связи ("Connection lost").

## Диагностика

- Проверьте правила UFW командой `ufw status numbered` и посмотрите, есть ли ограничение (`LIMIT`) для порта 22.
- Проверьте правила iptables командой `iptables -L -v -n` на наличие ограничений для порта 22.
- Изучите логи сервера (`/var/log/ufw.log`, `/var/log/iptables.log`, `/var/log/auth.log`, `/var/log/kern.log`) на предмет ошибок, связанных с брандмауэром.

## Решение

### Для UFW

- `ufw status numbered` — если порт 22 открыт, но с пометкой `LIMIT`, в этом и заключается проблема.
- `ufw delete <номер_правила>` — удалите правило. Скорее всего, у вас будут правила для IPv4 и IPv6, удалите оба.
  - После удаления снова проверьте статус `ufw status numbered`, чтобы убедиться, что правила удалены и вы знаете номера оставшихся правил.
- `ufw allow 22/tcp` — добавьте правило разрешения без лимита.

ИЛИ

- `ufw limit 22/tcp 100/minute` — установите более высокий лимит.

### Для пользователей Coolify Cloud

Вы можете разрешить входящие соединения с IP-адресов Coolify Cloud, добавив следующие правила в UFW для каждого адреса:

```sh
# Для каждого IPv4 адреса
ufw allow from <ipv4> to any port 22

# Для каждого IPv6 подсети
ufw allow from <ipv6> to any port 22
```

> Список актуальных IP-адресов Coolify Cloud всегда доступен по ссылкам: <https://coolify.io/ipv4.txt> и <https://coolify.io/ipv6.txt>
