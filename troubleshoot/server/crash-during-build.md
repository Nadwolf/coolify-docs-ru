# Сервер зависает или падает во время сборки

Coolify поддерживает два метода развертывания: развертывание с использованием **готового** Docker-образа и сборка из **исходного кода**. Понимание того, какой метод вы используете, поможет вам лучше решить проблемы с производительностью.

## 1. Понимание методов развертывания

### А. Развертывание готового Docker-образа

- Coolify запускает новый контейнер из существующего Docker-образа, который вы или кто-то другой уже собрали ранее.

### Б. Развертывание со сборкой из исходного кода

- Coolify собирает Docker-образ прямо на вашем сервере, используя исходный код вашего приложения, а затем запускает контейнер из этого свежесозданного образа.

## 2. Устранение проблем с производительностью

Если ваш сервер начинает сильно тормозить или падает во время развертывания, выполните следующие шаги в зависимости от вашего метода:

- **Для готовых образов:**
  - Поскольку контейнер запускается напрямую, высокое потребление ресурсов, скорее всего, связано с самой работой приложения.
  - **Решение:** Рассмотрите возможность апгрейда вашего сервера для соответствия потребностям приложения.

- **Для сборки из исходного кода:**
  - Процесс сборки Docker-образа (build) может перегрузить ваш сервер.
  - **Решение:** Перенесите процесс сборки на внешний [Build-сервер](/knowledge-base/server/build-server) или используйте альтернативные методы, такие как [GitHub Actions](https://docs.github.com/en/actions), чтобы сборка происходила вне вашего основного сервера. Также можно рассмотреть вариант увеличения мощностей текущего сервера.

- **Общий совет:**  
  Зайдите на сервер по SSH и запустите `htop`, чтобы следить за процессами. Выявите процесс, потребляющий слишком много ресурсов, и при необходимости завершите его.

## 3. Вынос сборки в GitHub Actions

Чтобы снизить нагрузку на сервер при развертывании, следуйте этому алгоритму:

- **Процесс:**
  - Используйте [GitHub Actions](https://docs.github.com/en/actions) для сборки Docker-образа вне сервера.
  - Пушите собранный образ в реестр контейнеров (например, [GitHub Container Registry](https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-container-registry)).
  - После завершения сборки настройте Coolify на автоматическое развертывание новой версии.
- **Преимущество:** Этот метод сводит к минимуму нагрузку на ваш сервер, обеспечивая более стабильное развертывание.

- **Узнать больше:** [Посмотрите наш пример workflow-файла для GitHub Actions](https://github.com/coollabsio/documentation-coolify/blob/main/.github/workflows/production-build.yml).

## Резюме

- **Методы развертывания:**
  - **Готовый образ:** Прямой запуск контейнера из существующего образа.
  - **Сборка из кода:** Сборка образа на вашем сервере перед запуском.
- **Устранение неполадок:**
  - Определите, вызвана ли проблема потребностями самого приложения или процессом сборки образа.
  - Улучшите характеристики сервера или вынесите сборку на другой ресурс по мере необходимости.
- **Оптимизация:**
  - Использование GitHub Actions для внешней сборки Docker-образов значительно снижает нагрузку на локальные ресурсы.
