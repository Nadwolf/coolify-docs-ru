# Сбои на Raspberry Pi

Если вы используете Raspberry Pi с объемом оперативной памяти всего 2 ГБ, вы можете столкнуться со сбоями системы даже при включенном файле подкачки (swap).

Это, скорее всего, связано с медленной работой SD-карт, часто используемых в Raspberry Pi, что может быть критично для стабильности.

## Решение
- Перейдите на модель Raspberry Pi с 4 ГБ оперативной памяти и выше для лучшей стабильности.
- Или ограничьте потребление памяти Docker, добавив следующую конфигурацию в файл `/etc/docker/daemon.json`:
  ```json
  {
    "memory": "1.8g"
  }
  ```
