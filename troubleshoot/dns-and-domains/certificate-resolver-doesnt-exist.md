# Резолвер сертификатов не найден в прокси Coolify

Если вы видите сообщение об ошибке `Cert resolver doesn't exist letsencrypt` в логах прокси Coolify, это обычно означает, что для вашего домена отсутствует SSL-сертификат. Эта проблема часто возникает, когда Coolify работает от имени пользователя без прав root.

## Симптомы
- Прокси Coolify не генерирует SSL-сертификат для вашего домена.
- В логах прокси появляется ошибка `Cert resolver doesn't exist letsencrypt`.

## Диагностика
Чтобы разобраться в ситуации, проверьте права доступа к файлу `acme.json` в директории `/data/coolify/proxy`, выполнив следующую команду:

```sh
ls -l /data/coolify/proxy
```

Вывод должен выглядеть примерно так:

```sh
-rwxr-x--- 1 user user 32757 Sep  3 15:39 acme.json
-rwxr-x--- 1 user user  1762 Sep  3 18:36 docker-compose.yml
drwxr-x--- 2 user user  4096 Sep  3 15:06 dynamic
```

## Первопричина
При использовании Coolify с учетной записью без прав root права доступа в директории `/data/coolify` могут быть настроены неверно.

Если права на файл `acme.json` **слишком свободные** (файл открыт для всех), Traefik откажется использовать его из соображений безопасности. Это не позволит Traefik записывать данные в файл и генерировать SSL-сертификаты.

С другой стороны, если права **слишком строгие**, Traefik не сможет прочитать файл или записать в него данные.

Трафику требуется строго определенный уровень разрешений для файла `acme.json`, чтобы всё работало корректно.

## Решение
Чтобы исправить это, выполните следующую команду для установки правильных прав доступа:

```sh
sudo chmod 600 /data/coolify/proxy/acme.json
```

### Что делает команда `sudo chmod 600`?
Команда `chmod 600` изменяет права доступа к файлу `/data/coolify/proxy/acme.json` так, что только владелец файла (учетная запись пользователя, используемая Coolify) может читать его и записывать в него данные. Это предотвращает доступ к файлу со стороны кого-либо еще, при этом позволяя Traefik использовать его по назначению.
