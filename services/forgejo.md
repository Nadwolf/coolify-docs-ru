## Что такое Forgejo?

Forgejo — это self-hosted легковесная "кузница программного обеспечения" (software forge). Она проста в установке, не требует сложного обслуживания и просто делает свою работу.

## Варианты развертывания

Forgejo доступен в четырех конфигурациях развертывания в Coolify:

### Forgejo (По умолчанию)
- **База данных:** SQLite (встроенная)
- **Сценарий использования:** Простые развертывания, тестирование или личный Git-хостинг
- **Компоненты:** Единый контейнер Forgejo со встроенной базой данных SQLite

### Forgejo с PostgreSQL
- **База данных:** PostgreSQL
- **Сценарий использования:** Prod-развертывания (production), требующие лучшей производительности и масштабируемости
- **Компоненты:**
  - Контейнер Forgejo
  - Контейнер PostgreSQL
  - Автоматическая настройка базы данных и проверки работоспособности (health checks)

### Forgejo с MySQL
- **База данных:** MySQL
- **Сценарий использования:** Prod-развертывания с предпочтением MySQL
- **Компоненты:**
  - Контейнер Forgejo
  - Контейнер MySQL
  - Автоматическая настройка базы данных и проверки работоспособности (health checks)

### Forgejo с MariaDB
- **База данных:** MariaDB
- **Сценарий использования:** Prod-развертывания с предпочтением MariaDB
- **Компоненты:**
  - Контейнер Forgejo
  - Контейнер MariaDB
  - Автоматическая настройка базы данных и проверки работоспособности (health checks)

## Forgejo Actions Runner

У Forgejo есть собственный (first party) "actions runner" для [выполнения задач (task jobs) в репозитории](https://forgejo.org/docs/latest/user/actions/), во многом похожий на [GitHub Actions](https://docs.github.com/en/actions) или [GitLab CI](https://docs.gitlab.com/ee/ci/index.html).

Coolify включает сервисы Forgejo с одним раннером (runner), использующим [Docker-in-Docker](https://hub.docker.com/_/docker) для обработки задач и отчетов по ним.

В связи со статусом альфа-версии раннера Forgejo, после первоначальной настройки требуется перезапуск (reboot) контейнера приложения Forgejo, чтобы полностью зарегистрировать общий секрет (shared secret) в Forgejo для валидации раннерами:

1. В разделе **Environment Variables** (Переменные окружения) конфигурации сервиса вы можете задать в качестве `RUNNER_SHARED_SECRET` случайную 40-символьную шестнадцатеричную строку. Команда `openssl rand -hex 20` создает то, что вы можете просто скопировать и вставить.
2. После успешной настройки Forgejo, **перезапустите сервис `forgejo`** и подождите несколько секунд, пока раннер (runner) не появится в разделе конфигурации _Actions_ в Forgejo.

Forgejo также совместим со сторонними CI-приложениями и платформами. Forgejo — это форк (fork) Gitea, поэтому инструкции по интеграции этих CI могут быть одинаковыми для обоих решений.
