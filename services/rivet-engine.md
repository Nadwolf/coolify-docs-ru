## Что такое Rivet Engine?

Rivet Engine — это бэкенд-движок, который обеспечивает работу Rivet Actors в масштабе. Это дополнительный компонент экосистемы Rivet, предоставляющий инфраструктуру корпоративного уровня (enterprise-grade) для запуск приложений с сохранением состояния (stateful workloads) с длительными процессами, устойчивым состоянием и возможностями работы в реальном времени.

Хотя вы можете использовать RivetKit (библиотеку TypeScript) с драйвером файловой системы или оперативной памяти для разработки, Rivet Engine предоставляет готовую к эксплуатации (production-ready) инфраструктуру для:

- **Автоматического масштабирования** от нуля до миллионов одновременно работающих акторов (actors)
- **Распределенного управления состоянием** с быстрым хранением состояния в оперативной памяти (in-memory)
- **Управления жизненным циклом акторов** с гибернацией и мгновенным пробуждением
- **Встроенной отказоустойчивости** с автоматическим переключением при сбоях и восстановлением
- **Edge-развертывания** для глобального распределения с низкой задержкой (low-latency)

Думайте об этом как о среде выполнения для продакшена (production runtime), которая берет ваших акторов RivetKit и легко масштабирует их в распределенной инфраструктуре.

### Основные возможности (Key Features)

- **Масштабирование для продакшена (Production-Ready Scaling)**: Автоматическое масштабирование от нуля до миллионов одновременных акторов без холодных запусков (cold starts).
- **Распределенное хранение состояния (Distributed State Storage)**: Состояние хранится на той же машине, что и вычисления, для сверхбыстрого чтения и записи без задержек базы данных.
- **Управление жизненным циклом акторов (Actor Lifecycle Management)**: Акторы автоматически переходят в режим гибернации при бездействии и мгновенно просыпаются по требованию, потребляя ресурсы только тогда, когда они активны.
- **Встроенная работа в реальном времени (Built-In Realtime)**: Нативная поддержка WebSockets и SSE для обновлений состояния и вещания в реальном времени.
- **Отказоустойчивая инфраструктура (Resilient Infrastructure)**: Автоматическое переключение при сбоях (failover) и восстановление с сохранением целостности состояния.
- **Готовность к Edge (Edge-Ready)**: Развертывание акторов близко к пользователям для мгновенного взаимодействия.
- **Поддержка нескольких сред выполнения (Multi-Runtime Support)**: Работает с Node.js, Bun, Deno и Cloudflare Workers.
- **Интеграция по API**: REST API для управления раннерами (runners), акторами и пространствами имен (namespaces).
- **Построено на веб-стандартах**: Использует стандартные протоколы HTTP, WebSockets и SSE.

### Варианты использования (Use Cases)

Rivet Engine обеспечивает работу приложений, требующих следующего:

- **ИИ-агенты (AI Agents)**: Устойчивые ИИ-помощники с постоянной памятью и возможностями потоковой передачи в реальном времени.
- **Совместная работа в реальном времени**: Совместные документы, доски и инструменты с использованием CRDT и синхронизацией в реальном времени.
- **Устойчивые рабочие процессы (Durable Workflows)**: Многошаговые рабочие процессы с автоматическим управлением состоянием и восстановлением.
- **Приложения Local-First**: Приложения, работающие в первую очередь в офлайне, с серверной синхронизацией и разрешением конфликтов.
- **Чат-боты и автоматизация**: Боты для Discord, Slack или автономные боты с сохранением состояния диалога.
- **Базы данных для каждого пользователя (Per-User Databases)**: Изолированные хранилища данных для каждого пользователя с доступом без задержек.
- **Многопользовательские игры**: Авторитетные игровые серверы с синхронизацией состояния в реальном времени.
- **Фоновая обработка**: Запланированные и повторяющиеся задачи без внешней инфраструктуры очередей.
- **Ограничение скорости (Rate Limiting)**: Распределенное ограничение скорости с использованием счетчиков и состояния в оперативной памяти.

### Как это работает

Rivet Engine предоставляет бэкенд-среду выполнения, которая:

1. **Управляет акторами**: Координирует жизненный цикл акторов, гибернацию и пробуждение в распределенной инфраструктуре.
2. **Обрабатывает состояние**: Хранит состояние акторов в оперативной памяти для быстрого доступа с созданием постоянных резервных копий в бэкендах хранения.
3. **Маршрутизирует запросы**: Эффективно направляет запросы клиентов к правильному экземпляру актора.
4. **Динамически масштабируется**: Автоматически создает и уничтожает экземпляры акторов в зависимости от спроса.
5. **Обеспечивает отказоустойчивость**: Контролирует состояние здоровья (health) и автоматически восстанавливает вышедшие из строя акторы с сохранением состояния.

### Интеграция с RivetKit

Rivet Engine бесшовно работает с RivetKit, библиотекой TypeScript для создания акторов:

- **Разработка**: Используйте RivetKit с драйверами файловой системы или оперативной памяти для локальной разработки.
- **Продакшн**: Разверните в Rivet Engine для автоматического масштабирования и использования корпоративных функций.
- **Самостоятельный хостинг (Self-Hosted)**: Запускайте Rivet Engine на собственной инфраструктуре для полного контроля.
- **Управляемый сервис (Managed)**: Используйте Rivet Cloud для развертывания в 1 клик на управляемой инфраструктуре.

### Варианты развертывания

Разверните Rivet Engine на:

- **Собственном хостинге** (Coolify, Docker, Kubernetes)
- **Облачных платформах** (Vercel, Railway, AWS ECS, Google Cloud Run, Hetzner)
- **Rivet Cloud** (Управляемый сервис, развертывание в 1 клик)

### Бэкенды хранения (Storage Backends)

Rivet Engine поддерживает несколько бэкендов хранения для состояния акторов:

- PostgreSQL
- Файловая система (File System)
- Оперативная память (Memory) (для разработки/тестирования)
- Пользовательские адаптеры хранения (Custom storage adapters)
