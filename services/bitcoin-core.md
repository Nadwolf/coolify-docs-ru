# Bitcoin Core

<ZoomableImage src="/docs/images/services/bitcoin.svg" alt="Дашборд Bitcoin Core" />

## Что такое Bitcoin Core

Self-hosted полная нода (full node) Bitcoin Core.

## Публичная нода vs. Частное использование

По умолчанию сервис Bitcoin Core в Coolify настроен для частного/внутреннего использования. Это означает, что он предназначен для подключения кошельков или приложений с того же сервера или частной сети, и **не принимает входящие P2P-соединения из общедоступного интернета**. Это настройка по умолчанию, ориентированная на безопасность (снижает поверхность атаки, особенно для новых пользователей).

### Как запустить публичную ноду

Если вы хотите, чтобы ваша биткоин-нода была полностью публичным узлом (peer) и принимала входящие соединения от других нод:

1. **Откройте P2P-порт в вашем файле Compose:**  
   Добавьте следующее под вашим сервисом `bitcoin-core`:   
    ```text
    ports:
    - "8333:8333"
    ```

2. **Откройте порт 8333 в брандмауэре (firewall) вашего сервера:**  
- На Ubuntu с UFW:
  ```bash
  sudo ufw allow 8333/tcp
  ```
- На CentOS с firewalld:
  ```bash
  sudo firewall-cmd --add-port=8333/tcp --permanent
  sudo firewall-cmd --reload
  ```
- Для облачных серверов обновите группу безопасности (security group) или правила брандмауэра вашего провайдера, чтобы разрешить входящие соединения по TCP на порт 8333.

### Что следует учесть

- **Поведение по умолчанию:**  
Без секции `ports` ваша нода не будет принимать входящие P2P-соединения, но по-прежнему сможет создавать исходящие соединения и полностью синхронизировать блокчейн.
- **Безопасность:**  
То, что порт 8333 не открыт по умолчанию, помогает защитить пользователей, которые могут не понимать риски запуска публичной биткоин-ноды.
- **Продвинутые пользователи (Advanced Users):**  
Если вы понимаете риски и хотите внести больший вклад в сеть Bitcoin, выполните шаги выше, чтобы запустить публичную ноду.
