## Что такое WordPress?

WordPress — это бесплатная система управления контентом с открытым исходным кодом, написанная на PHP и работающая в паре с базой данных MySQL/MariaDB.
Она используется для создания веб-сайтов, блогов и приложений.

## Варианты развертывания

WordPress доступен в двух конфигурациях развертывания в Coolify:

### WordPress с MariaDB
- **База данных:** MariaDB
- **Сценарий использования:** Решения для продакшена с предпочтением MariaDB (рекомендуется для большинства пользователей)
- **Компоненты:**
  - Контейнер WordPress
  - Контейнер MariaDB
  - Автоматическая настройка базы данных и проверки работоспособности (health checks)

### WordPress с MySQL
- **База данных:** MySQL
- **Сценарий использования:** Решения для продакшена с предпочтением MySQL
- **Компоненты:**
  - Контейнер WordPress
  - Контейнер MySQL
  - Автоматическая настройка базы данных и проверки работоспособности (health checks)

Оба варианта обеспечивают эквивалентную функциональность — выбирайте тот, который соответствует вашим предпочтениям в отношении базы данных или существующей инфраструктуре.

## Часто задаваемые вопросы (FAQ)

### Как увеличить лимит размера загружаемого файла?

Вы можете увеличить лимит размера загрузки, выполнив следующие шаги:

1. Откройте файл `.htaccess` через терминал Coolify (или через SSH, выполнив `docker exec -ti container_id /bin/sh`).
2. Добавьте следующие строки в конец файла с помощью `vi` или `nano`:

```php
# END WordPress - эта строка уже есть в файле

php_value upload_max_filesize 256M
php_value post_max_size 256M
php_value max_execution_time 300
php_value max_input_time 300
```

3. Сохраните и закройте файл.
4. Обновите веб-сайт в браузере. Изменения должны примениться автоматически.

### Как исправить циклическую переадресацию (Redirection Loop) в WordPress?

Если ваш сайт на WordPress застрял в цикле перенаправлений, выполните следующие действия для решения проблемы:

Доступ к файлу .htaccess:

1. Откройте файл `.htaccess` через терминал Coolify (или через SSH, выполнив `docker exec -ti container_id /bin/sh`).
2. Перейдите в корневой каталог WordPress и найдите файл .htaccess. Отредактируйте файл .htaccess, добавив следующий блок:

```
<IfModule mod_setenvif.c>
  SetEnvIf X-Forwarded-Proto "^https$" HTTPS
</IfModule>
```

3. Сохраните и закройте файл.
4. Обновите веб-сайт в браузере. Изменения должны примениться автоматически.
