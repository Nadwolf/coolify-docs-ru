Nixpacks — это билдпак с открытым исходным кодом, созданный [Railway](https://railway.com?utm_source=coolify.io). Исходный код доступен на [Github](https://github.com/railwayapp/nixpacks?utm_source=coolify.io). Coolify использует Nixpacks в качестве одного из основных билдпаков.

Nixpacks проверяет ваш git-репозиторий (который в терминах nixpacks называется «**source directory**» — исходная директория) и генерирует Dockerfile, на основе которого затем собирается Docker-образ.

С помощью Nixpacks можно развертывать как полностью статические веб-сайты, так и динамические приложения. Как только ваш репозиторий будет готов, вы сможете легко развернуть проект в Coolify.

## Как использовать Nixpacks?

В Coolify Nixpacks можно использовать только для развертываний на основе Git.

### 1. Создайте новый ресурс в Coolify

В дашборде Coolify откройте свой проект и нажмите кнопку **Create New Resource**.

<ZoomableImage src="/docs/images/builds/packs/nixpacks/1.webp" alt="скриншот дашборда Coolify" />

### 2. Выберите вариант развертывания

<ZoomableImage src="/docs/images/builds/packs/nixpacks/2.webp" alt="скриншот дашборда Coolify" />

**A.** Если ваш Git-репозиторий публичный, выберите вариант **Public Repository**.

**B.** Если ваш репозиторий приватный, вы можете выбрать **Github App** или **Deploy Key**. (Эти методы требуют дополнительной настройки. Вы можете ознакомиться с руководствами по настройке [Github App](/applications/ci-cd/github/integration#with-github-app-recommended) или [Deploy Key](/applications/ci-cd/github/integration#with-deploy-keys).)

### 3. Выберите ваш Git-репозиторий

Если вы используете публичный репозиторий, вставьте URL вашего репозитория GitHub при появлении запроса. Шаги практически одинаковы для всех остальных вариантов.

<ZoomableImage src="/docs/images/builds/packs/nixpacks/3.webp" alt="скриншот дашборда Coolify" />

### 4. Выберите билдпак

Coolify по умолчанию предложит использовать Nixpacks. Если этого не произошло, выберите Nixpacks вручную.

<ZoomableImage src="/docs/images/builds/packs/nixpacks/4.webp" alt="скриншот дашборда Coolify" />

### 5. Настройте билдпак

Доступны различные параметры, такие как базовая директория (Base Directory), публичная директория (Publish Directory) и порты. Они немного различаются в зависимости от типа деплоя (статический сайт или приложение). Ниже описаны два варианта развертывания через Nixpacks.

- [Как развернуть полностью статический сайт](#как-развернуть-полностью-статический-сайт)
- [Как развернуть динамический сайт или приложение](#как-развернуть-динамический-сайт-или-приложение)

## Как развернуть полностью статический сайт?

Сначала выполните шаги из предыдущего раздела: [Как использовать Nixpacks](#как-использовать-nixpacks). После этого следуйте приведенным ниже инструкциям:

<ZoomableImage src="/docs/images/builds/packs/nixpacks/5.webp" alt="скриншот дашборда Coolify" />

1. **Branch (Ветка):** Coolify автоматически определит ветку из вашего репозитория.

2. **Base Directory (Базовая директория):** введите директорию, которую Nixpacks должен использовать как корень (например, `/`, если ваши файлы находятся в корне, или путь к подпапке).
   - Если у вас монорепозиторий, вы можете указать путь к конкретной папке (например, `/frontend`).

3. **Is it a static Site? (Это статический сайт?):** включите эту опцию, чтобы активировать статический режим.

4. **Port (Порт):** после включения режима статического сайта порт будет автоматически установлен в значение `80` и его нельзя будет изменить (это сделано намеренно).

5. **Publish Directory (Публичная директория):** после включения опции «Is it a static Site» этот параметр станет видимым. Вам нужно указать директорию, в которую генерируются статические файлы (обычно это `/dist`).

6. Нажмите кнопку **Continue**, когда все данные будут заполнены верно.

7. Выберите веб-сервер для вашего статического сайта:

<ZoomableImage src="/docs/images/builds/packs/nixpacks/6.webp" alt="скриншот дашборда Coolify" />

- Начиная с Coolify **v4.0.0-beta.404**, единственным доступным вариантом веб-сервера является [Nginx](https://nginx.org/en/?utm_source=coolify.io). Поэтому **Nginx** будет выбран по умолчанию.

8. Нажмите кнопку **Deploy**. Процесс развертывания обычно занимает меньше минуты.

9. Настройте конфигурацию веб-сервера <Badge type="warning" text="Опционально" />

<ZoomableImage src="/docs/images/builds/packs/nixpacks/7.webp" alt="скриншот дашборда Coolify" />

- Coolify предоставляет стандартную конфигурацию веб-сервера, подходящую для большинства случаев.
- Если вы хотите изменить её, нажмите кнопку **Generate**, чтобы загрузить настройки по умолчанию и внести правки.

::: warning ВНИМАНИЕ!
Вам необходимо нажать кнопку **Restart**, чтобы новая конфигурация вступила в силу.
:::

### Как это работает?

Nixpacks соберет сайт на основе вашего кода и создаст Docker-образ с веб-сервером. Это означает, что в итоговом образе будет веб-сервер, готовый отдавать ваши HTML, CSS и JavaScript файлы.

## Как развернуть динамический сайт или приложение?

Сначала выполните шаги из раздела: [Как использовать Nixpacks](#как-использовать-nixpacks). Затем следуйте этой инструкции:

<ZoomableImage src="/docs/images/builds/packs/nixpacks/8.webp" alt="скриншот дашборда Coolify" />

1. **Branch (Ветка):** Coolify автоматически определит ветку из вашего репозитория.

2. **Base Directory (Базовая директория):** введите директорию, которую Nixpacks должен использовать в качестве корня.
   - Для монорепозиториев укажите путь к папке приложения (например, `/backend`).

3. **Port (Порт):** введите порт, по которому ваше приложение ожидает входящие запросы.

4. **Is it a static Site?:** оставьте эту опцию выключенной, так как вы развертываете динамическое приложение.

5. Нажмите кнопку **Continue**, когда все параметры будут настроены.

6. После этого вы сможете уточнить такие настройки, как домен и переменные окружения, а затем нажать **Deploy** для запуска приложения.

### Как это работает?

Nixpacks анализирует ваш код, собирает Docker-образ и запускает контейнер на его основе.

## Продвинутые настройки

### Переменные окружения

Вы можете настраивать поведение Nixpacks с помощью переменных окружения. Существует множество переменных для различных фреймворков, подробную информацию можно найти в документации: [Nixpacks Environment Variables](https://nixpacks.com/docs/configuration/environment?utm_source=coolify.io).

Чтобы добавить или изменить переменные окружения в Coolify, перейдите на вкладку **Environment Variables**.

<ZoomableImage src="/docs/images/builds/packs/nixpacks/9.webp" alt="скриншот дашборда Coolify" />

---

### Команды (Commands)

При необходимости вы можете переопределить стандартные команды установки (install), сборки (build) и запуска (start). Прокрутите вниз до раздела сборки в Coolify и введите свои команды.

<ZoomableImage src="/docs/images/builds/packs/nixpacks/10.webp" alt="скриншот дашборда Coolify" />

::: warning Примечание:
Для того чтобы эти изменения вступили в силу, вам может потребоваться файл `nixpacks.toml` в вашем репозитории.
:::

---

### Файл конфигурации

Nixpacks поддерживает указание настроек сборки в файлах `nixpacks.toml` или `nixpacks.json`. Если такой файл присутствует в корне вашего репозитория, он будет использован автоматически. Подробности см. в [документации Nixpacks](https://nixpacks.com/docs/configuration/file?utm_source=coolify.io).

## Известные проблемы

### Устаревшие пакеты/зависимости

Иногда Nixpacks может использовать более старые версии пакетов, чем вам требуется. Это часто случается с Node.js, где может понадобиться конкретная минорная версия.

Для фиксации версии Node.js см. руководство [Node.js Versioning](/applications/build-packs/nixpacks/node-versioning).

Для других пакетов вы можете обновить версию архива `nixpkgs` в вашем файле `nixpacks.toml`. Подробнее об этом в документации Nixpacks: [nixpkgs archive](https://nixpacks.com/docs/configuration/file#nixpkgs-archive?utm_source=coolify.io).
