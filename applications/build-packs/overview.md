Coolify развертывает каждое приложение в виде Docker-контейнера. Это означает, что ваше приложение запускается в собственном изолированном контейнере.

Чтобы запустить контейнер, вам необходим Docker-образ, собранный из вашего исходного кода.

Билдпаки (Build packs) помогают создавать этот Docker-образ и управлять процессом сборки и развертывания.


## Зачем использовать билдпаки?

- **Упрощение процесса сборки:** некоторые билдпаки автоматически создают Docker-образ, необходимый для деплоя, поэтому вам не нужно тратить время на изучение того, как писать Dockerfile самостоятельно.

- **Гибкость для разных проектов:** поскольку все приложения разные, вы можете выбрать билдпак, который лучше всего подходит для ваших нужд — будь то автоматизированное решение или кастомная конфигурация.


## Как работают билдпаки

Каждый билдпак предлагает свой подход к сборке Docker-образа:

- **Автоматическое создание Dockerfile:** билдпаки вроде Nixpacks и Static Build Pack автоматически генерируют Dockerfile на основе вашей кодовой базы и собирают образ.
  - Это позволяет быстро развернуть приложение, не создавая Dockerfile вручную.

- **Пользовательский Dockerfile или Docker Compose:** билдпаки Dockerfile и Docker Compose позволяют использовать файлы Dockerfile или Docker Compose, которые уже есть в вашем коде.
  - Это дает вам полный контроль над тем, как собирается ваш образ и как взаимодействуют между собой несколько сервисов.


## Выбор подходящего билдпака

В Coolify есть четыре основных билдпака для различных требований:

- **Nixpacks:** отлично подходит для быстрого и автоматического создания образов с минимальной настройкой.

- **Static Build Pack:** идеален для статических сайтов и простых приложений, которым не требуется серверная обработка.

- **Dockerfile:** используйте собственный Dockerfile, если хотите иметь полный контроль над процессом сборки образа.

- **Docker Compose:** идеально подходит для развертывания сложных многокомпонентных приложений с помощью вашего файла Docker Compose.
