# Интеграция с Gitea

Это руководство покажет вам, как использовать репозитории на базе Gitea с Coolify.

## Публичные репозитории

Вы можете использовать публичные репозитории без какой-либо дополнительной настройки.

1. Выберите опцию `Public repository` в Coolify при создании нового ресурса.
2. Добавьте URL вашего репозитория в поле ввода, например: `https://gitea.com/heyandras/coolify-examples`

::: warning Внимание
Вы можете использовать только HTTPS URL.
:::

3. Это всё! Coolify автоматически загрузит последнюю версию вашего репозитория и развернет её.

## Приватные репозитории

Настройка приватных репозиториев требует нескольких дополнительных шагов.

1. Добавьте приватный ключ (он же `Deploy Key`) в Coolify и в настройки вашего репозитория Gitea в меню `Repository Settings` / `Access Keys`.

::: warning Внимание
  - Вы можете сгенерировать новую пару ключей с помощью следующей команды: 
  
  ```bash
  ssh-keygen -t rsa -b 4096 -C "deploy_key" 
  ```

  - Или вы можете позволить Coolify создать новый ключ для вас в меню `Keys & Tokens`.
:::

2. Создайте новый ресурс и выберите `Private Repository (with deploy key)`.
3. Добавьте SSH URL вашего репозитория в поле ввода, например: `git@gitea.com:heyandras/coolify-examples.git`

::: warning Внимание
Вам необходимо использовать SSH URL, который начинается с `git@`.
:::

4. Это всё! Coolify автоматически загрузит последнюю версию вашего репозитория и развернет её.

## Автоматическое развертывание коммитов через вебхуки (Опционально)

Вы можете добавить кастомный URL вебхука в ваш репозиторий Gitea, чтобы запускать новое развертывание при пуше в репозиторий.

::: warning Внимание
Это можно настроить как для публичных, так и для приватных репозиториев.
:::

В настройках вашего ресурса есть меню `Webhooks`. В разделе `Manual Git Webhooks` вы найдете URL, который нужно прописать в настройках Gitea.

1. Установите секретный ключ в поле `Gitea Webhook Secret`.
2. Перейдите в ваш репозиторий в Gitea и откройте меню `Repository Settings` / `Webhooks` в разделе `Repository hooks`.
3. Добавьте URL из Coolify в поле `URL` и укажите секретный токен.
4. Выберите опцию `Push`.
5. Готово! Теперь при пуше в репозиторий Gitea отправит запрос вебхука в Coolify, что запустит новое развертывание.

## Развертывания для Merge Request через вебхуки (Опционально)

Вы можете добавить кастомный URL вебхука в ваш репозиторий Gitea, чтобы запускать новое развертывание при создании нового Pull Request (в Gitea это Pull Requests).

::: warning Внимание
Это можно настроить как для публичных, так и для приватных репозиториев.
:::

Процесс такой же, как и в предыдущем пункте. В меню `Repository Settings` / `Webhooks` вам нужно выбрать следующие события в разделе `Pull Request`:

- `Created`
- `Updated`
- `Merged`
- `Declined`
