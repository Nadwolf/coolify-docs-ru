# Другие Git-провайдеры

Это руководство покажет вам, как использовать других Git-провайдеров с Coolify, таких как Gogs, Forgejo и любую другую Git-совместимую платформу.

## Публичные репозитории

Вы можете использовать публичные репозитории от любого Git-провайдера без какой-либо дополнительной настройки.

1. Выберите опцию `Public repository` в Coolify при создании нового ресурса.
2. Добавьте URL вашего репозитория в поле ввода, например: `https://git.example.com/username/repository`

::: warning Внимание
Вы можете использовать только HTTPS URL.
:::

3. Это всё! Coolify автоматически загрузит последнюю версию вашего репозитория и развернет её.

## Приватные репозитории

Для аутентификации в приватных репозиториях требуются ключи развертывания (deploy keys).

### С использованием ключей развертывания (Deploy Keys)

1. Добавьте приватный ключ (он же `Deploy Key`) в Coolify и в настройки вашего Git-репозитория (обычно находится в разделе `Settings` / `Deploy Keys` или `SSH Keys`).

::: warning Внимание

- Вы можете создать новую пару ключей с помощью следующей команды:

```bash
ssh-keygen -t ed25519 -C "coolify_deploy_key"
```

- Вы также можете позволить Coolify создать новый ключ для вас в меню `Keys & Tokens`.
:::

2. Создайте новый ресурс и выберите `Private Repository (with deploy key)`.
3. Добавьте SSH URL вашего репозитория в поле ввода, например: `git@git.example.com:username/repository.git`

::: warning Внимание
Вам необходимо использовать SSH URL, который начинается с `git@`.
:::

4. Это всё! Coolify автоматически загрузит последнюю версию вашего репозитория и развернет её.

## Автоматическое развертывание коммитов (Опционально)

Для Git-провайдеров без прямой интеграции автоматическое развертывание требует инициации через эндпоинт Deploy Webhook.

::: warning Внимание
Это требует, чтобы ваш Git-провайдер поддерживал автоматизацию рабочих процессов или действия через вебхуки (похожие на GitHub Actions).

Если ваш провайдер этого не поддерживает, вам придется запускать развертывание вручную через дашборд Coolify.
:::

### Предварительные условия

1. Создайте [Coolify API Token](/api-reference/authorization) в дашборде Coolify.
2. Получите URL Deploy Webhook вашего ресурса (Ваш ресурс → меню `Webhooks` → `Deploy Webhook`).

### Настройка через Workflow/CI систему

Если ваш Git-провайдер поддерживает автоматизацию (например, GitHub Actions, GitLab CI, Forgejo Actions и т. д.), вы можете настроить автоматический запуск деплоя:

1. Добавьте ваш API-токен Coolify в секреты репозитория (например, `COOLIFY_TOKEN`).
2. Добавьте URL Deploy Webhook в секреты репозитория (например, `COOLIFY_WEBHOOK`).
3. Создайте файл рабочего процесса, который запускается при событии push:

```yaml
# Пример рабочего процесса (синтаксис зависит от провайдера)
on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Trigger Coolify Deployment
        run: |
          curl --request GET "${{ secrets.COOLIFY_WEBHOOK }}" \
            --header "Authorization: Bearer ${{ secrets.COOLIFY_TOKEN }}"
```

4. Готово! Теперь при пуше в репозиторий рабочий процесс отправит запрос в Coolify для запуска нового развертывания.

::: tip Альтернатива: Прямые вебхуки
Некоторые Git-провайдеры позволяют вебхукам отправлять кастомные заголовки. Если это поддерживается, вы можете настроить вебхук для отправки GET-запроса с заголовком `Authorization: Bearer ВАШ_ТОКЕН` напрямую на URL Deploy Webhook, без использования файлов workflow.
:::

## Поддерживаемые Git-провайдеры

Этот метод работает с любым Git-провайдером, поддерживающим стандартные протоколы Git, включая:

- Gogs
- Forgejo
- Свои инстансы GitLab CE/EE
- Кастомные Git-серверы (gitolite и др.)
- Любые платформы, совместимые с Git-over-SSH
