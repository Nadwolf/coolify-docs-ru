# CI/CD с Git-провайдерами

Приложения в Coolify разработаны для развертывания непосредственно из **Git-репозиториев**, что позволяет реализовать рабочие процессы непрерывной интеграции и непрерывного развертывания (CI/CD). Это означает, что ваши приложения автоматически обновляются при пуше изменений кода в репозиторий.

## Как работает интеграция с Git

Когда вы развертываете приложение в Coolify, вы подключаете его к Git-репозиторию от **любого Git-провайдера**. Coolify работает со всеми популярными Git-платформами, включая:

- **[GitHub](/applications/ci-cd/github/integration)** — полная интеграция через GitHub App или ключи развертывания (deploy keys)
- **[GitLab](/applications/ci-cd/gitlab/integration)** — интеграция с GitLab через вебхуки
- **[Bitbucket](/applications/ci-cd/bitbucket/integration)** — интеграция с Bitbucket через вебхуки
- **[Gitea](/applications/ci-cd/gitea/integration)** — self-hosted Git-платформа
- **Любой другой Git-провайдер** — работает с любой Git-совместимой платформой с публичными репозиториями или через ключи развертывания

После подключения Coolify:

1. **Загружает ваш исходный код** из репозитория.
2. **Собирает Docker-образ**, используя выбранный вами [билдпак (build pack)](/applications/build-packs/overview).
3. **Развертывает контейнер** на вашем сервере.
4. **Следит за изменениями** и автоматически выполняет повторное развертывание при пуше новых коммитов (если включен автодеплой).

## Ключевые преимущества развертывания на основе Git

### Автоматические развертывания

Просто отправьте код в свой репозиторий, и Coolify автоматически соберет и развернет ваше приложение. Ручное вмешательство не требуется.

### Предварительные развертывания (Preview Deployments)

Тестируйте Pull Requests в изолированных средах перед слиянием с основной веткой. Каждый PR получает свой уникальный URL.

### Интеграция с системой контроля версий

- Отслеживайте историю развертываний вместе с коммитами кода.
- Легко откатывайтесь к предыдущим версиям.
- Видьте точно, какая версия кода сейчас работает в продакшене.

### Рабочие процессы CI/CD

- Интегрируйте с GitHub Actions, GitLab CI и другими инструментами CI.
- Запускайте тесты перед развертыванием.
- Автоматизируйте сложные конвейеры (pipelines) развертывания.

::: tip Альтернатива: Деплой без Git
Если вы хотите развернуть свое приложение **без подключения к Git-провайдеру**, вы можете развернуть его как [Сервис (Service)](/services/introduction). Сервисы позволяют:

- Загружать файл Docker Compose напрямую в Coolify.
- Развертывать из Docker-образов без исходного кода.
- Управлять приложением вручную без интеграции с Git.

Это полезно в сценариях, когда вы собираете Docker-образы в другом месте или предпочитаете ручной контроль над развертываниями.
:::

## Методы доступа к репозиторию

Coolify поддерживает несколько способов доступа к вашим Git-репозиториям:

### Публичные репозитории

Просто укажите HTTPS URL вашего публичного репозитория. Аутентификация не требуется. Работает с любым Git-провайдером.

### Приватные репозитории

Выберите метод аутентификации в зависимости от вашего Git-провайдера:

1. **Интеграция через приложение Git-провайдера (Рекомендуется для поддерживаемых платформ)**

   - Доступно для GitHub.
   - Полная интеграция с автоматическими вебхуками.
   - Развертывание превью для Pull Request.
   - Обновление статусов коммитов.
   - Не требуется управление SSH-ключами.

2. **Ключи развертывания (Deploy Keys) — работает с любым Git-провайдером**
   - Аутентификация на основе SSH.
   - Универсальная поддержка — работает с любой Git-платформой.
   - Требуется более детальная ручная настройка вебхуков.
   - Лучше подходит для закрытых или ограниченных сред.
   - Идеально для кастомных или self-hosted Git-серверов.

## Поддерживаемые Git-провайдеры

Хотя мы предоставляем подробные руководства по интеграции для популярных платформ, **Coolify работает с любым Git-провайдером**, поддерживающим стандартные протоколы Git:

- **Публичные репозитории**: любой Git-провайдер (аутентификация не требуется).
- **С интеграцией через приложение**: GitHub.
- **С ключами развертывания**: любой Git-провайдер (GitHub, GitLab, Bitbucket, Gitea, Gogs, Forgejo, self-hosted решения и другие).
