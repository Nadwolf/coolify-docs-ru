# Интеграция с GitLab

## Публичные репозитории

Вы можете использовать публичные репозитории без какой-либо дополнительной настройки.

1. Выберите опцию `Public repository` в Coolify при создании нового ресурса.
2. Добавьте URL вашего репозитория в поле ввода, например: `https://gitlab.com/andrasbacsai/coolify-examples`

   ::: warning Внимание
   Вы можете использовать только HTTPS URL.
   :::

3. Это всё! Coolify автоматически загрузит последнюю версию вашего репозитория и развернет её.

## Приватные репозитории

Для настройки приватных репозиториев требуется несколько дополнительных шагов.

1. Добавьте приватный ключ (он же `Deploy Key`) в Coolify и в настройки вашего репозитория GitLab (меню `Settings` / `Repository` / `Deploy Keys`).

::: warning Внимание

- Вы можете сгенерировать новую пару ключей с помощью следующей команды:

```bash
ssh-keygen -t rsa -b 4096 -C "deploy_key"
```

- Вы также можете позволить Coolify создать новый ключ для вас в меню `Keys & Tokens`.
:::

2. Создайте новый ресурс и выберите `Private Repository (with deploy key)`.
3. Добавьте SSH URL вашего репозитория в поле ввода, например: `git@gitlab.com:andrasbacsai/coolify-examples.git`

::: warning Внимание
Вам необходимо использовать SSH URL, начинающийся с `git@`.
:::

4. Это всё! Coolify автоматически загрузит последнюю версию вашего репозитория и развернет её.

## Публичный реестр контейнеров (Container Registry)

Вы можете использовать публичный реестр контейнеров без дополнительной настройки.

1. Выберите опцию `Docker Image` в Coolify при создании нового ресурса.
2. Добавьте URL вашего публичного реестра контейнеров вместе с тегом в поле ввода, например: `registry.gitlab.com/username/repository:latest`.
3. Нажмите кнопку `Deploy`.
4. Это всё! Coolify автоматически загрузит последнюю версию из реестра и развернет её.

## Приватный реестр контейнеров (Container Registry)

Для настройки приватного реестра контейнеров требуется несколько дополнительных шагов.

1. Создайте `Deploy Token` в вашем репозитории GitLab в меню `Settings` / `Repository` / `Deploy Token` с разрешением `read_registry`. В результате вы получите логин (`username`) и токен (`token`).
2. Авторизуйтесь в Docker с этими учетными данными на вашем сервере Coolify:

```bash
echo "token-xxx" | docker login registry.privategitlab.com -u gitlab+deploy-token-xxx --password-stdin
```

3. Выберите опцию `Docker Image` в Coolify при создании нового ресурса.
4. Добавьте URL вашего приватного реестра контейнеров вместе с тегом в поле ввода, например: `registry.privategitlab.com/username/repository:latest`.
5. Нажмите кнопку `Deploy`.
6. Это всё! Coolify автоматически загрузит последнюю версию из реестра и развернет её.

## Автоматическое развертывание коммитов через вебхуки (Опционально)

Вы можете добавить кастомный URL вебхука в ваш репозиторий GitLab, чтобы запускать новое развертывание при пуше в репозиторий.

::: warning Внимание
Это можно настроить как для публичных, так и для приватных репозиториев.
:::

В настройках вашего ресурса есть меню `Webhooks`. В разделе `Manual Git Webhooks` вы найдете URL, который нужно прописать в настройках GitLab.

1. Установите секретный ключ в поле `GitLab Webhook Secret`.
2. Перейдите в ваш репозиторий в GitLab и откройте меню `Settings` / `Webhooks`.
3. Добавьте URL из Coolify в поле `URL` и укажите секретный токен.
4. Выберите опцию `Push events`.
5. Готово! Теперь при пуше в репозиторий GitLab отправит запрос вебхука в Coolify, что запустит новое развертывание.

## Развертывания для Merge Request через вебхуки (Опционально)

Вы можете добавить кастомный URL вебхука в ваш репозиторий GitLab, чтобы запускать новое развертывание при создании нового Merge Request.

::: warning Внимание
Это можно настроить как для публичных, так и для приватных репозиториев.
:::

Процесс такой же, как и в предыдущем пункте, но вам нужно выбрать опцию `Merge request events` в меню `Settings` / `Webhooks`.
