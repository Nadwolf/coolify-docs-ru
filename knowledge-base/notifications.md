# Уведомления

Coolify предоставляет мощную систему уведомлений с поддержкой множества каналов связи. Вы можете настроить их во вкладке **Notifications** в панели управления Coolify.

## Провайдеры уведомлений

Ниже приведены руководства по настройке поддерживаемых каналов связи.

### Email (Электронная почта)

::: info
Уведомления по электронной почте можно настроить через любой SMTP-сервер или сервис Resend.
:::

1. Перейдите в раздел **Notifications** → **Email**.

2. Выберите провайдера:
   - Использовать общесистемные (транзакционные) настройки (если вы используете Coolify Cloud, эта опция уже настроена за вас; в self-hosted версии она настраивается в Instance Settings).
   - Пользовательский SMTP-сервер.
   - Resend.

3. Выполните настройку в зависимости от выбранного варианта:

#### Общесистемные настройки (System-Wide)

- Включите чекбокс `Use system wide (transactional) email settings`.
- Если вы настраиваете эти параметры самостоятельно, обратитесь к разделу [Настройка SMTP-сервера](#настройка-smtp-сервера) ниже.

#### Настройка SMTP-сервера

- Заполните следующие поля:
  - `From Name` — имя отправителя.
  - `From Address` — email-адрес, с которого будут приходить уведомления.
  - `Host` — адрес вашего SMTP-сервера (например, `smtp.mail.com`).
  - `Port` — порт SMTP:
    - Порт **587** — порт StartTLS (наиболее распространенный).
    - Порт **465** — порт TLS/SSL (рекомендуется для максимальной безопасности).
  - `Username` — имя пользователя для авторизации на SMTP-сервере.
  - `Password` — пароль для авторизации.
  - `Encryption` — выберите метод шифрования:
    - **StartTLS** — начинает соединение без шифрования, затем переходит на безопасный протокол TLS (обычно используется с портом 587).
    - **TLS/SSL** — использует шифрование с самого начала установки соединения (обычно используется с портом 465 — в Coolify автоматически добавляет префикс `ssl://` перед хостом).
    - **None** — без шифрования (НЕ рекомендуется, так как это небезопасно).
  - `Timeout` — время ожидания соединения в секундах.
- Активируйте SMTP-сервер, поставив чекбокс `Enabled`.

::: success Совет
Мы рекомендуем использовать шифрование TLS/SSL с портом 465 для наиболее безопасного соединения.
:::

::: info
Некоторые хостинг-провайдеры накладывают ограничения на порты. Например, Hetzner по умолчанию блокирует порт 465 для новых аккаунтов. Уточните доступность портов у вашего провайдера.
:::

#### Настройка Resend

- Введите ваш `Resend API Key`.
- Включите отправку через Resend (чекбокс).

После настройки нажмите `Send Test Email`, чтобы проверить, правильно ли всё работает.

### Telegram

1. **Первоначальная настройка (только через телефон)**
   - Создайте аккаунт в Telegram, если у вас его еще нет.
   - Откройте приложение на телефоне.
   - Перейдите в **Settings** (Настройки) → **Profile** (Профиль).
   - Установите `username` (имя пользователя) — это рекомендуется для стабильной работы.

2. **Создание бота**
   - Напишите боту [@BotFather](https://t.me/botfather).
   - Отправьте команду `/newbot`.
   - Следуйте инструкциям для создания бота.
   - Скопируйте полученный `Bot Token`.

::: info
После копирования токена удалите сообщение с ним из чата. Храните токен в безопасности, так как он дает полный контроль над вашим ботом.
:::

3. **Добавление бота в группу**
   - Создайте новую группу в Telegram.
   - Добавьте в неё вашего бота (найдите его по имени, которое вы выбрали при создании).

4. **Назначение бота администратором**
   - Нажмите на название группы вверху чата.
   - Откройте список участников.
   - Нажмите на имя бота.
   - Выберите **Promote to admin** (Назначить администратором) или **Add to group or channel** → включите переключатель администратора.

5. **Включение тем/топиков (опционально, только через телефон)**
   - Откройте группу на телефоне.
   - Нажмите на название группы -> Изменить (иконка карандаша).
   - Найдите и включите функцию **Topics**.
   - Теперь вы можете создавать отдельные темы для структурирования уведомлений.

6. **Получение необходимых ID**
   - Перейдите в браузере по ссылке: `https://api.telegram.org/bot{ВАШ_ТОКЕН_БОТА}/getUpdates` (замените `{ВАШ_ТОКЕН_БОТА}` на реальный токен).
   - Отправьте любое тестовое сообщение в группу (или в конкретную тему, если вы их включили).
   - Обновите страницу в браузере и найдите в ответе следующие значения:
     - `Chat ID`: ищите строку `"chat": {"id": -100XXXXXXXXX,` (ID групп обычно начинаются с `-100`).
     - `Thread/Topic ID` (если используете темы): ищите строку `"message_thread_id":XXXXX,`.

7. **Настройка в Coolify**
   - Перейдите в **Notifications** → **Telegram**.
   - Введите `bot token`.
   - Введите `chat ID` (включая знак минуса `-`, если он есть).
   - Введите `thread/topic ID` (только если используете темы).
   - Сохраните настройки.
   - Включите канал Telegram (чекбокс `Enabled`).
   - Нажмите `Test notification`, чтобы проверить связь.

::: info
Распространенные проблемы:
- Убедитесь, что бот назначен администратором группы.
- Не забудьте знак минуса в Chat ID.
- Если `getUpdates` возвращает пустой ответ, отправьте еще одно сообщение в группу и снова обновите страницу.
:::

### Discord

1. **Подготовка сервера и канала**
   - Создайте сервер (Server) в Discord или используйте существующий.
   - Создайте текстовый канал (Text Channel) специально для уведомлений Coolify.

2. **Создание вебхука**
   - Откройте настройки сервера Discord (Server Settings).
   - Перейдите в раздел **Integrations** → **Webhooks**.
   - Нажмите **New Webhook**.
   - Выберите созданный канал в качестве целевого.
   - Скопируйте `webhook URL`.

3. **Настройка в Coolify**
   - Перейдите в **Notifications** → **Discord**.
   - Вставьте скопированный `webhook URL`.
   - Сохраните и включите канал.
   - Нажмите `Test notification` для проверки.

### Slack / Mattermost

#### Использование Slack

1. **Создание приложения Slack**
   - Перейдите на страницу [Slack API](https://api.slack.com/apps).
   - Нажмите **Create New App** → **From scratch**.
   - Выберите ваше рабочее пространство (Workspace).

2. **Включение вебхуков**
   - Перейдите в раздел **Incoming Webhooks**.
   - Переключите ползунок **Activate Incoming Webhooks** в положение On.
   - Нажмите кнопку **Add New Webhook to Workspace**.
   - Выберите канал, в который будут приходить уведомления, и скопируйте сгенерированный `webhook URL`.

3. **Настройка в Coolify**
   - Перейдите в **Notifications** → **Slack**.
   - Вставьте `webhook URL`.
   - Сохраните настройки и включите канал.

#### Использование с Mattermost

Вебхуки в формате Slack, которые отправляет Coolify, полностью совместимы с Mattermost.

1. **Создание вебхука в Mattermost**
   - В меню Mattermost перейдите в **Integrations** → **Incoming Webhooks**.
   - Нажмите **Add Incoming Webhook**.
   - Укажите имя, выберите канал, сохраните настройки.
   - Скопируйте сгенерированный URL-адрес (он будет иметь формат: `https://your-mattermost-server.com/hooks/xxx-key-xxx`).

2. **Настройка в Coolify**
   - В интерфейсе Coolify используйте раздел **Notifications** → **Slack** и вставьте туда URL, полученный от Mattermost.

### Pushover (Push-уведомления на телефон)

1. **Получение User Key**
   - Зарегистрируйтесь на сайте [Pushover](https://pushover.net).
   - Скопируйте ваш `User Key`, расположенный в верхней правой части страницы в личном кабинете.

2. **Создание приложения**
   - Перейдите на страницу [Create Application](https://pushover.net/apps/build).
   - Заполните необходимые данные и создайте приложение.
   - Скопируйте полученный `API Token/Key`.

3. **Настройка в Coolify**
   - Перейдите в **Notifications** → **Pushover**.
   - Введите ваш `User Key` и `API Key` приложения.
   - Сохраните настройки и включите этот канал связи.

### Webhook (Пользовательские вебхуки)

Позволяет отправлять уведомления на любой сторонний API или микросервис.

1. **Настройка в Coolify**
   - Перейдите в **Notifications** → **Webhook**.
   - Введите целевой URL (HTTP или HTTPS).
   - Сохраните конфигурацию и включите канал.

Coolify будет автоматически отправлять `POST`-запросы (с JSON-payload) на указанный адрес при возникновении событий.

## События для уведомлений

Вы можете гибко настроить, какие именно события будут активировать отправку уведомлений в каждый конкретный канал:

### Развертывания (Deployments)
- Успешное завершение развертывания.
- Ошибка в процессе развертывания.
- Изменения статуса контейнера (остановка, перезапуск).

### Резервное копирование (Backups)
- Успешное создание бэкапа.
- Ошибка при создании бэкапа.

### Запланированные задачи
- Успешное выполнение задачи (Scheduled Tasks).
- Ошибка выполнения задачи.

### Сервер (Отслеживание состояния серверов)
- Успешная автоматическая очистка Docker.
- Ошибка при очистке Docker.
- Критический уровень использования диска.
- Доступность сервера (Сервер доступен/Сервер недоступен).
- Информации о применении патчей (Server Patching).
- Уведомления об устаревшей версии прокси Traefik.

::: success Разделение потоков
Вы можете настроить разные типы событий для разных каналов. Например: отправлять все уведомления об ошибках на Email для гарантированной доставки, а статус успешных деплоев — в общий чат Slack или Telegram для команды.
:::
