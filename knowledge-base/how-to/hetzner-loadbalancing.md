# Балансировка нагрузки на Hetzner
Hetzner Cloud — это облачный провайдер, предлагающий широкий спектр услуг. В этом руководстве мы покажем, как развернуть приложение с балансировкой нагрузки с помощью Coolify в Hetzner Cloud.

## Введение

Ваше приложение растет, и вам нужно его масштабировать. Один из самых распространенных способов масштабирования — использование балансировщика нагрузки (Load Balancer). Он распределяет входящий трафик между несколькими серверами, гарантируя, что ни один сервер не будет перегружен. Это помогает повысить производительность и надежность вашего приложения.

В этом руководстве мы будем использовать два (или более) сервера для хостинга приложения и один балансировщик для распределения трафика. Мы также покажем, как настроить отдельный сервер базы данных, если он вам необходим.

::: success Совет
Вся инфраструктура обойдется примерно в ~15€ в месяц.
:::

## Требования

- Доменное имя, управляемое через Hetzner Cloud.
- 2 (или более) сервера в Hetzner Cloud.
- Балансировщик нагрузки (Load Balancer) в Hetzner Cloud.
- Настроенный Firewall для ограничения доступа к серверам.
- Опционально: сервер базы данных в Hetzner Cloud.

## Шаги

1. ### Покупка серверов (Buy Servers)

    Сначала купите два (или более) сервера в Hetzner Cloud. Выбирайте тип сервера и локацию в зависимости от ваших нужд. Убедитесь, что ресурсов серверов достаточно для работы приложения.

    ::: warning Осторожно
    Обязательно создайте и подключите **частную сеть** (Private Network) к вашим серверам. Это позволит им взаимодействовать друг с другом по внутренним IP-адресам.
    :::

    Если у вас еще нет аккаунта в Hetzner Cloud, вы можете зарегистрироваться по этой [реферальной ссылке](https://coolify.io/hetzner).

2. ### Добавление и валидация серверов в Coolify

    Добавьте купленные серверы в Coolify, нажав кнопку `Add Server` в дашборде и следуя инструкциям. Coolify автоматически установит на них необходимое программное обеспечение.

3. ### Опционально: Развертывание базы данных

    Предпочтительно иметь отдельный сервер для базы данных — не только ради производительности, но и для безопасности. Так вы сможете подключать несколько приложений к одному централизованному серверу БД.

    Пропустите этот шаг, если база данных вам не нужна или она размещена в другом месте.

    Добавьте сервер БД в Coolify и разверните нужный тип базы (MySQL, PostgreSQL и т.д.).

    ::: warning Осторожно
    В этом руководстве мы не используем обратный прокси для БД. Мы привяжем базу данных к публичному IP сервера, но ограничим доступ с помощью брандмауэра (Firewall).

    Вы можете отключить прокси во вкладке `Server`, выбрав тип прокси `None`.
    :::

    Убедитесь, что `Ports Mappings` настроены правильно, чтобы серверы приложений могли достучаться до БД:
    - PostgreSQL: `5432:5432`
    - MySQL/MariaDB: `3306:3306`
    - MongoDB: `27017:27017`

4. ### Развертывание приложений (Deploy your applications)

    Разверните приложение на серверах. Процесс зависит от типа вашего стека. В Coolify вы можете привязать более одного сервера к одному приложению.

    ::: warning Осторожно
    Мы не будем использовать обратный прокси Coolify на серверах приложений, так как маршрутизацией займется балансировщик Hetzner.

    Отключите прокси во вкладке `Server`, выбрав `None`.
    :::

    Настройте `Ports Mappings` так, чтобы порты приложения пробрасывались на публичный IP сервера.

    Если приложению нужна БД, обновите строку подключения, указав адрес сервера базы данных.
    ::: success Совет
     Hetzner Cloud позволяет серверам общаться по частной сети. Это безопаснее и быстрее. Используйте частные IP в строке подключения к БД.
    :::

    Проверьте работоспособность приложения, зайдя на публичный IP сервера и соответствующий порт в браузере.

5. ### Добавление и настройка балансировщика нагрузки

    Теперь добавим балансировщик для распределения трафика.

    Он будет выполнять следующие задачи:
    - Распределять входящий трафик между серверами.
    - Терминировать TLS-соединения (обеспечивать HTTPS и передавать трафик на серверы по HTTP).
    - Проверять состояние (health checks) серверов и направлять трафик только на "здоровые" узлы.

    Допустим, ваше приложение работает на порту `8000` на обоих серверах, и у вас есть домен `example.com`.

    1. Перейдите в Hetzner Cloud Console и выберите `Load Balancers` в боковой панели.
    2. Нажмите `Create Load Balancer`.
    3. Выберите тот же регион, где находятся ваши серверы.
    4. Выберите частную сеть, созданную для серверов.
    5. В разделе `Targets` выберите все ваши серверы приложений. **Важно: выберите `(use the private network)`.**
    6. В разделе `Services` удалите стандартный сервис и добавьте `TLS Termination`.
    7. В поле `Destination Port` укажите `8000` (исходящий порт `443`).
    8. Добавьте сертификат для вашего домена (Let's Encrypt будет сгенерирован автоматически).
    9. В разделе `Health Check` укажите эндпоинт вашего приложения, который возвращает `200 OK`.
    10. Опционально: включите `Sticky sessions`, если нужно, чтобы пользователь всегда попадал на один и тот же сервер.
    11. Выберите алгоритм балансировки.
    12. Назовите балансировщик и нажмите `Create & Buy now`.

    Дождитесь, пока статус балансировщика станет "healthy". После этого проверьте доступность по IP балансировщика.

6. ### Настройка DNS

    Направьте ваш домен на IP балансировщика.

    1. Зайдите в настройки DNS вашего провайдера.
    2. Добавьте `A`-запись для домена, указав IP-адрес балансировщика.
    3. Дождитесь обновления DNS-записей.

    После этого ваше приложение должно быть доступно по доменному имени.

7. ### Настройка брандмауэра (Setup Firewall)

    Hetzner Cloud позволяет ограничить доступ к серверам.

    Так как балансировщик общается с серверами по частной сети, нам нужно защитить только публичные IP.

    Входящие (`Inbound`) правила разрешают трафик. Всё остальное блокируется по умолчанию.

    1. В консоли Hetzner выберите `Firewalls` слева.
    2. Нажмите `Create Firewall`.
    3. Добавьте правило для SSH-доступа только со своего IP-адреса.
    4. В разделе `Apply rule` добавьте ваши серверы.
    5. Назовите брандмауэр и создайте его.

    ::: success Совет
    Опционально: вы можете добавить дополнительные правила для разрешения доступа к специфическим портам или адресам.
    :::
