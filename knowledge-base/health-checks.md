# Проверки состояния (Health checks)

Проверки состояния — это способ убедиться, что ваши приложения и сервисы работают корректно. Они позволяют Coolify отслеживать состояние ваших ресурсов и гарантировать, что трафик направляется только на работающие (здоровые) экземпляры. Это крайне важно, например, для корректной работы [скользящих обновлений (Rolling Updates)](/knowledge-base/rolling-updates).

## Traefik

Если в качестве обратного прокси используется Traefik, проверки состояния являются критически важной частью процесса маршрутизации трафика к вашим ресурсам.

### Включено

Если для ресурса *включены* проверки состояния, Traefik будет направлять на него трафик только в том случае, если проверка пройдена успешно. Если проверка не проходит, Traefik не будет перенаправлять трафик на этот ресурс.

**В браузере это будет выглядеть как ошибка `404 Not Found` или `No available server`.**

::: tip Устранение неудачных проверок состояния
Если вы столкнулись с ошибкой "No available server", ознакомьтесь с нашим подробным [руководством по устранению неполадок](/troubleshoot/applications/no-available-server), где описаны основные причины и способы их решения.
:::

### Выключено

Если для ресурса *выключены* проверки состояния, Traefik будет направлять на него трафик независимо от реального состояния приложения.

## Настройка проверок состояния

### Приложения

Существует два способа настройки проверок состояния для ваших приложений:

<ZoomableImage src="/docs/images/knowledge-base/resources/healthcheck.webp" alt="Настройка Healthcheck" />

1. **Через интерфейс (UI)**: Вы можете настроить проверку прямо в панели управления Coolify при создании или редактировании приложения. Вы можете указать путь для проверки, ожидаемый код HTTP-ответа и интервал проверок. Для выполнения такой проверки внутри контейнера **должен быть установлен `curl` или `wget`**.

2. **Через Dockerfile**: Вы также можете определить проверку прямо в вашем `Dockerfile`, используя инструкцию [HEALTHCHECK](https://docs.docker.com/engine/reference/builder/#healthcheck). Она позволяет задать произвольную команду для запуска и условия успешного прохождения проверки.

Если проверки определены и в интерфейсе Coolify, и в `Dockerfile`, приоритет будет отдан инструкции из **Dockerfile**.

### Стек сервисов

Для сервисов или приложений, использующих [билдпак Docker Compose](/applications/build-packs/docker-compose), проверки состояния должны быть определены либо в `Dockerfile` каждого отдельного сервиса, либо в файле `docker-compose.yaml` с помощью атрибута [healthcheck](https://docs.docker.com/reference/compose-file/services/#healthcheck).

::: tip Когда стоит использовать проверки состояния?
Настоятельно рекомендуется включать проверки для всех ресурсов. Это единственный надежный способ гарантировать, что пользовательский трафик получают только исправно работающие экземпляры приложения.

Однако, если по какой-то причине вы не можете настроить проверку корректно (например, в образе нет нужных утилит), вы можете её временно отключить. Но помните: в этом случае трафик будет идти на ресурс, даже если приложение внутри контейнера зависло или упало.
:::
