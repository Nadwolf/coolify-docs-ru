# Смена ключа Localhost
Это руководство покажет вам, как заменить сохраненный приватный ключ localhost для Coolify.

::: warning Осторожно
  Сделайте резервную копию приватного ключа в базе данных перед началом этой операции.
:::

## Удаление ключа из базы данных
В базе данных Postgres, где хранятся данные Coolify, есть таблица `private_keys`.
Эта таблица содержит приватные ключи для всех серверов, подключенных к инстансу Coolify.
Будьте осторожны, чтобы случайно не удалить не те ключи, так как обычно это действие нельзя отменить.

1. Перейдите к таблице `private_keys` в базе данных Postgres для Coolify.
   - Эта таблица находится в схеме `public`.

2. Выберите строку, помеченную как:
   - Индекс строки и ID равны 0, а поле name — "localhost's key".

3. Удалите выбранную строку из базы данных.
   - После успешного удаления перейдите к следующему разделу.

::: success Совет
  Удобнее всего делать это через графический интерфейс управления базой данных.
:::

## Установка новых ключей на хост-машине

После того как ключ был удален из базы данных, вам нужно добавить новые ключи в правильное место в папках данных Coolify.

1. Полностью остановите ваш инстанс Coolify.

2. Найдите директорию `/ssh/keys` вашей установки Coolify.
   - Обычно она находится в `/data/ssh/keys`, если вы использовали быструю установку.

3. Запишите публичный и приватный ключи в соответствующие файлы:
   - Ваш публичный ключ ED25519 должен храниться в `id.root@host.docker.internal.pub`.
   - Ваш приватный ключ ED25519 должен храниться в `id.root@host.docker.internal`.

4. Запустите инстанс Coolify.
   - Coolify теперь перенесет ваши ключи в базу данных Postgres.
   - Убедитесь, что сидер `Database\Seeders\PopulateSshKeysDirectorySeeder` отработал без ошибок.

::: info Примечание
  Если вы получили сообщение "SSH key found for the Coolify host machine (localhost)", операция прошла успешно, и если нет других уведомлений, ваш новый ключ должен работать.
:::
