# Настройка DNS
В общем случае вам нужна `A`-запись для всех доменов или поддоменов, которые вы хотите использовать. Она должна указывать на IP-адрес сервера, где вы планируете развернуть приложение.

Обратите внимание, что вы можете использовать один и тот же IP-адрес для множества доменов и поддоменов.

**В примерах ниже `1.1.1.1` — это IP-адрес вашего сервера.**

## Один домен

Допустим, вы хотите развернуть ресурс на `example.com` с IP-адресом `1.1.1.1`.

- Вам нужно создать `A`-запись для `example.com`, указывающую на `1.1.1.1`.

::: success Совет
  Вы также можете добавить `www.example.com` в качестве `A`-записи и настроить перенаправление (redirect) внутри Coolify с помощью выбранного обратного прокси (reverse proxy).
:::

Теперь вы можете использовать `https://example.com` как FQDN (полное доменное имя) для любого вашего ресурса, включая саму панель Coolify.

## Wildcard-домены (подстановочные)

Допустим, вы хотите развернуть ресурс на `*.example.com` с IP-адресом `1.1.1.1`.

- Вам нужно создать `A`-запись для `*.example.com`, указывающую на `1.1.1.1`.

Это позволит вам использовать любой поддомен `example.com` в качестве FQDN для ваших ресурсов.

Например, вы сможете использовать `https://app.example.com` или `https://api.example.com`.

## Автогенерируемые домены

Если вы настроили wildcard-домен в вашей DNS-панели, Coolify может автоматически создавать домены для ваших ресурсов.
Вам просто нужно зайти в настройки сервера (`Server settings`) и установить дочерний домен в поле `Wildcard Domain`, например `https://example.com`.

Теперь при создании нового ресурса:

- Вы получите случайный поддомен для вашего приложения, например `https://random.example.com`.
- Это также касается превью-развертываний (Preview Deployments), например `https://<PRId>.random.example.com`.

::: success Совет
  Шаблон URL для превью можно изменить на странице приложения во вкладке `Preview Deployments`.
:::

## Домен инстанса (панели)

Если вы используете self-hosted версию Coolify, вы можете задать кастомный домен для панели управления на странице `/settings`.
