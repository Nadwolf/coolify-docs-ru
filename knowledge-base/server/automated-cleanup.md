# Автоматическая очистка Docker

В Coolify встроена функция автоматической очистки Docker, чтобы предотвратить переполнение диска на серверах. Это руководство объясняет, как настроить эту функцию и что именно она делает.

## Настройка

Вы можете настроить автоматическую очистку в разделе:
`Servers` > `ВАШ_СЕРВЕР` > `Configuration` > `Advanced`

### Доступные параметры

1. **Порог очистки Docker (Docker Cleanup Threshold)**
   - Устанавливает процент заполнения диска, при котором запускается очистка.
   - Пример: если установлено 80%, очистка начнется, когда диск будет заполнен более чем на 80%.

2. **Частота очистки Docker (Docker Cleanup Frequency)**
   - Позволяет планировать очистку с помощью [выражений cron](/knowledge-base/cron-syntax), если включена опция `Force Docker Cleanup`.

::: success Совет
- Мы рекомендуем включить `Force Docker Cleanup` и настроить расписание очистки через cron.
- Это обеспечит более надежную работу системы по сравнению с ожиданием достижения порога заполнения диска.
:::

3. **Дополнительные опции очистки**
   - Очистка неиспользуемых томов (volumes) — (внимание: это может привести к потере данных).
   - Очистка неиспользуемых сетей (networks).

## Как это работает

### Меры безопасности
- Если в данный момент идет развертывание (deployment), очистка не будет запущена. Это предотвращает удаление образа, который используется прямо сейчас.
- Затрагиваются только ресурсы, которыми управляет Coolify.

### Процесс очистки
При запуске (по расписанию или по достижении порога) система выполняет следующие действия:

- Удаляет остановленные контейнеры, управляемые Coolify (без потери данных, так как сами контейнеры не являются местом хранения постоянных данных).
- Удаляет неиспользуемые Docker-образы.
- Очищает кэш сборки Docker.
- Удаляет старые версии вспомогательных образов Coolify.
- Удаляет неиспользуемые тома Docker (если включено).
- Удаляет неиспользуемые сети Docker (если включено).
