# Введение
Независимо от типа вашего сервера (локальный или удаленный), он должен соответствовать следующим требованиям:

- **Связь (Connectivity)**
  - Наличие SSH-соединения между Coolify и сервером с аутентификацией по SSH-ключам.
     ::: success Совет
     Ваш публичный ключ должен быть добавлен в файл `~/.ssh/authorized_keys` пользователя **root**.
     Если у вас нет SSH-ключа, вы можете сгенерировать его прямо в Coolify одной кнопкой или создать вручную.
     :::
- **Docker Engine (версии 24+)**

## Типы серверов
- **Localhost**: сервер, на котором установлен сам Coolify.
- **Remote Server**: любой удаленный Linux-сервер.

## Localhost
Чтобы иметь возможность управлять сервером, на котором запущен Coolify, Docker-контейнер Coolify должен иметь доступ к хост-серверу через SSH.
Вы можете использовать localhost как сервер для запуска всех ваших ресурсов, но это не рекомендуется: высокая нагрузка от приложений может замедлить или нарушить работу самой панели Coolify.

::: success Совет
  Вы можете использовать нашу [облачную версию](https://app.coolify.io), и тогда вам понадобится сервер только для ваших приложений.
  В облачную версию включены дополнительные возможности, такие как бесплатные уведомления по email, хранилище S3 и др., в зависимости от вашего тарифного плана.
:::

## Удаленный сервер (Remote Server)
Вы можете подключить к Coolify любые типы серверов: VPS, Raspberry Pi или даже ноутбук с Linux.

::: success Совет
Если у вас еще нет сервера или провайдера, мы рекомендуем Hetzner.
Вы можете воспользоваться нашей [реферальной ссылкой](https://coolify.io/hetzner). Это поможет нам поддерживать проект.
:::

### Туннели Cloudflare (Cloudflare Tunnels)
Вы также можете настроить использование туннелей Cloudflare для ваших серверов.

::: success Совет
Coolify не устанавливает `cloudflared` на ваш сервер автоматически, это нужно сделать заранее.
Панель лишь добавляет правильную команду `ProxyCommand` (`ProxyCommand <ip / hostname> access ssh --hostname %h`) ко всем SSH-соединениям.
:::

## Конфигурация с несколькими серверами
При настройке нескольких серверов в Coolify важно понимать, как работает маршрутизация трафика:

- **Независимая работа прокси**: На каждом сервере запускается собственный прокси-сервер, который обрабатывает входящие запросы для приложений, развернутых именно на этом сервере.
- **Прямой поток трафика**: Трафик для приложений на дополнительных серверах идет напрямую на эти серверы, а не через основной сервер Coolify.
- **Настройка DNS**: Вам нужно направлять DNS-записи домена на IP-адрес того сервера, где развернуто приложение, а не на основной сервер Coolify.

### Роль основного сервера
Основной сервер Coolify:
- Предоставляет интерфейс управления (UI) для контроля приложений на всех подключенных серверах.
- Устанавливает SSH-соединения с дополнительными серверами для деплоя и управления.
- Выполняет проверки состояния (health checks) и мониторинг.
- **НЕ** проксирует и не перенаправляет трафик к приложениям на дополнительных серверах.

Такая архитектура снижает задержки и повышает производительность приложений, исключая лишние сетевые переходы.

### Порог очистки диска (Disk Cleanup threshold)
Подробнее об автоматической очистке и порогах использования диска читайте в разделе [Автоматическая очистка](/knowledge-base/server/automated-cleanup).

### Wildcard-домен
Вы можете установить wildcard-домен (например: `http://example.com`) для вашего сервера. Это позволит автоматически назначать домены всем ресурсам, подключенным к этому серверу.

Пример: UUID вашего приложения — `vgsco4o`.
Если настроен wildcard-домен, вы получите FQDN: `http://vgsco4o.example.com`.
Если wildcard-домен не задан, Coolify сгенерирует домен [sslip.io](https://sslip.io) — это бесплатный сервис, который можно использовать везде.
В этом случае адрес будет: `http://vgsco4o.127.0.0.1.sslip.io`, где `127.0.0.1` — IP вашего сервера.

::: success Совет
При использовании нескольких серверов помните, что домен каждого приложения должен указывать на конкретный сервер, где это приложение развернуто.
:::

## Прокси
- **Traefik**: Автоматическая настройка Traefik (v2) на основе ваших развернутых ресурсов.
- **Custom/None**: Вы настраиваете прокси вручную (только для продвинутых пользователей).

::: success Совет
  Скоро мы добавим поддержку Nginx и Caddy с полностью автоматизированной настройкой.
:::

### Traefik
Coolify использует прокси Traefik по умолчанию для создания обратного прокси для ваших ресурсов.

::: success Совет
  Traefik запускается только в том случае, если вы не выбрали другой прокси для сервера и у вас настроен домен для ресурса или для самого инстанса Coolify.
:::

#### Динамическая конфигурация
Вы всегда можете добавить свою собственную конфигурацию в настройки прокси через интерфейс Coolify (`/server/<server_uuid>/proxy`).
