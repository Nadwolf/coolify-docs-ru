# Несколько серверов (Multiple Servers)
С помощью этой функции вы можете развернуть одно и то же приложение на нескольких серверах, поставить перед ними балансировщик нагрузки и получить высокодоступное приложение.

::: danger ВНИМАНИЕ!
**Это экспериментальная функция.**
:::

## Требования

- Каждый сервер должен быть добавлен в Coolify, валидирован и доступен.
- Каждый сервер (включая опциональный сервер сборки) должен иметь одинаковую архитектуру (AMD64 или ARM).
- Вы должны пушить собранный образ в Docker Registry (реестр контейнеров). Coolify автоматизирует этот процесс, вам нужно лишь [авторизоваться в реестре](/knowledge-base/docker/registry#docker-credentials) на сервере.

## Как использовать?

При настройке (или если уже настроено) приложения вы выбрали сервер/сеть для деплоя. Это будет ваш основной сервер.

Все дополнительные серверы должны быть указаны в меню `Servers`, просто выберите их.

Теперь каждый раз, когда вы передеплоите, перезапускаете или останавливаете приложение, действие будет выполнено на всех серверах.

Если деплой требует процесса сборки, он будет запущен на основном сервере (или на сервере сборки, если он есть). После сборки образ будет загружен в Docker Registry, и только после этого все остальные серверы получат уведомление о необходимости скачать (pull) и развернуть этот образ.

## Как настроить балансировщик нагрузки?

На данный момент этот процесс не автоматизирован, поэтому вам нужно настроить балансировщик вручную. Есть два способа.

### Маппинг портов на хост (Port mapping to host)

Если вы настроили `Ports Mappings` для вашего приложения (сопоставили порт контейнера с портом сервера), вам нужно:

1. Указать все комбинации `IP:PORT` ваших серверов в качестве назначений в балансировщике.
2. Удалить все домены из поля `Domains` в Coolify.

В этом случае прокси Coolify (Proxy) не используется, так как приложение доступно напрямую по IP:PORT.

::: success Совет
  Это очень простой и эффективный способ. Но помните: вам следует разрешить входящие соединения на выбранный порт `PORT` только с IP балансировщика, иначе любой желающий сможет зайти в ваше приложение напрямую, минуя балансировщик.
:::

### Использование домена

В этом случае вам нужно указать **домен балансировщика с протоколом HTTP (не HTTPS)** в поле `Domains`. Затем настройте балансировщик с терминированием SSL (SSL termination).

Такая конфигурация позволяет использовать несколько доменов с одним балансировщиком.
