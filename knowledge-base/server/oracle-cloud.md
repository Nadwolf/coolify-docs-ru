# Oracle Cloud
Если вы используете бесплатную версию `Oracle Cloud ARM server`, вам необходимо выполнить несколько дополнительных шагов для работы с Coolify (причем как для самого инстанса Coolify, так и для использования его в качестве удаленного сервера).

## Обычный пользователь (Normal user)
Использование пользователя без прав root находится в `экспериментальном` режиме и работает через `sudo`.
- Убедитесь, что SSH-ключ добавлен в файл `~/.ssh/authorized_keys` конкретного пользователя.
- Вся конфигурация установлена для sudo. Подробности [здесь](/knowledge-base/server/non-root-user).

## Настройка пользователя Root (Setup Root User)

По умолчанию вы не можете войти под пользователем root. Чтобы включить его, выполните следующие действия:

1. Переключитесь на пользователя root: `sudo su -`
2. Отредактируйте `/etc/ssh/sshd_config` и измените `PermitRootLogin` на `without-password`.
3. Перезапустите службу SSH: `service sshd restart`
4. Добавьте публичный ключ в файл `/root/.ssh/authorized_keys` (этот же ключ должен быть добавлен и в ваш инстанс Coolify).

## Правила брандмауэра (Firewall Rules)

Это требуется только в том случае, если вы сами размещаете Coolify на ARM-сервере Oracle (self-host).

По умолчанию в Oracle ARM включен брандмауэр, и вам нужно разрешить определенные порты для использования Coolify.

Подробности смотрите на [этой](/knowledge-base/server/firewall) странице.
