<br />

Сервер сборки позволяет компилировать ваши проекты отдельно от сервера, на котором работает ваше живое приложение.

Это помогает снизить нагрузку на хостинг-сервер и гарантирует, что производительность вашего приложения не пострадает во время процесса сборки.

## Требования
Перед настройкой сервера сборки убедитесь, что:

- Итоговые образы пушатся в реестр контейнеров (container registry).
- Сервер сборки авторизован в этом реестре. Подробнее см. в [этом руководстве](/knowledge-base/docker/registry).
- Сервер сборки имеет доступ к вашему исходному коду.
- На сервере сборки установлен Docker Engine.
- Архитектура сервера сборки совпадает с архитектурой серверов развертывания.

::: success Совет:
Если у вас несколько серверов сборки, Coolify выберет один из них случайным образом.
:::

## Как использовать сервер сборки
Чтобы начать использовать сервер сборки в Coolify, выполните следующие шаги:

1. **Добавьте новый сервер в Coolify:**
   В дашборде Coolify перейдите на страницу серверов и нажмите кнопку **+ Add**.

   <ZoomableImage src="/docs/images/builds/servers/1.webp" alt="скриншот дашборда Coolify" />

   - Если вы уже подключили сервер к Coolify, можете пропустить этот шаг.

2. **Включите функцию сервера сборки:**
   В модальном окне активируйте опцию **Build Server**.
   <ZoomableImage src="/docs/images/builds/servers/2.webp" alt="скриншот дашборда Coolify" />

   - Если сервер уже подключен, включите функцию Build Server, как показано на изображении ниже.
   <ZoomableImage src="/docs/images/builds/servers/3.webp" alt="скриншот дашборда Coolify" />

::: warning ВНИМАНИЕ!
Начиная с **Coolify v4.0.0-beta.408**, вы не можете развертывать приложения на сервере, который помечен как сервер сборки (build server).

Если вы хотите развертывать на нем приложения, снимите галочку "build server" в настройках этого сервера в дашборде Coolify.

   <ZoomableImage src="/docs/images/builds/servers/3.webp" alt="скриншот дашборда Coolify" />
:::

## Настройка ресурса для использования сервера сборки
Чтобы назначить сервер сборки существующему ресурсу, выполните следующие действия:

1. **Перейдите к вашему ресурсу:**
   В дашборде Coolify перейдите в общие настройки (General Settings) приложения, для которого вы хотите использовать сервер сборки.

2. **Активируйте опцию сервера сборки:**
   В разделе **Build** включите опцию `Use a Build Server ?`.

   <ZoomableImage src="/docs/images/builds/servers/4.webp" alt="скриншот дашборда Coolify" />

3. **Настройте реестр контейнеров:**
   Убедитесь, что ваш сервер сборки авторизован в реестре контейнеров. Подробнее см. в [этом руководстве](/knowledge-base/docker/registry).
