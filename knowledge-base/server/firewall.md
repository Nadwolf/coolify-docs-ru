# Брандмауэр (Firewall)
Для корректной работы Coolify в различных средах требуется открыть определенные сетевые порты. Эти порты обеспечивают веб-доступ, SSH-соединения, сессии терминала и связь в реальном времени.

Список необходимых портов может немного отличаться в зависимости от того, используете ли вы self-hosted версию или управляемую ([Coolify Cloud](https://coolify.io/pricing/)).

## Coolify Self-hosted
Для обеспечения полной функциональности при самостоятельном размещении Coolify необходимо открыть следующие порты:

* **8000** — HTTP-доступ к дашборду Coolify.
* **6001** — Связь в реальном времени (WebSockets).
* **6002** — Доступ к терминалу (требуется для Coolify версии 4.0.0-beta.336 и выше).
* **22** — SSH-доступ (или ваш кастомный SSH-порт).
* **80** — Генерация SSL-сертификатов через обратный прокси (Traefik или Caddy).
* **443** — HTTPS-трафик.

Эти порты обязательны, если вы заходите в Coolify напрямую по IP-адресу сервера (например, `http://<SERVER_IP>:8000`).

::: success Совет
Если вы используете кастомный домен с интегрированным обратным прокси Coolify (Traefik или Caddy), вы можете безопасно закрыть порты **8000**, **6001** и **6002** после того, как настроите доступ к дашборду через ваш домен.
:::

::: warning Осторожно
  Если вы используете `Oracle Cloud Free ARM Server`, вам необходимо разрешить эти порты в панели управления Oracle, иначе вы не сможете получить доступ к инстансу из интернета после установки.
:::

## Coolify Cloud
Для серверов, подключенных к Coolify Cloud, должны быть открыты следующие порты:

* **22** — SSH-доступ (или ваш кастомный SSH-порт).
* **80** — Генерация SSL-сертификатов через обратный прокси (Traefik или Caddy).
* **443** — HTTPS-трафик.

Это единственные необходимые порты, так как все остальные сервисы управляются для вас через Coolify Cloud.

## Закрытие портов с помощью брандмауэра
Coolify работает на Docker, который использует правила iptables на основе NAT. Эти правила могут обходить традиционные брандмауэры Linux, такие как UFW. В результате блокировка портов только с помощью UFW может оказаться неэффективной.

### Рекомендуемый подход
Большинство облачных провайдеров предлагают встроенные брандмауэры в своих панелях управления. Если ваш провайдер поддерживает это, **настоятельно рекомендуется использовать настройки их брандмауэра** для управления открытыми портами вместо локальных инструментов вроде UFW.

Если ваш провайдер не предоставляет такой функционал, вы можете воспользоваться одним из следующих продвинутых методов:

### Coolify Self-hosted
::: danger ВНИМАНИЕ!!
  Неправильное изменение настроек брандмауэра может привести к потере доступа к серверу, который будет трудно восстановить.
  
  Выполняйте следующие шаги **только в случае необходимости** и если вы полностью понимаете последствия.
:::

#### Использование `ufw-docker`
[ufw-docker](https://github.com/chaifeng/ufw-docker) — это инструмент, поддерживаемый сообществом, который помогает связать UFW и Docker, позволяя эффективно блокировать определенные порты. Ознакомьтесь с [репозиторием на GitHub](https://github.com/chaifeng/ufw-docker) для получения полных инструкций по установке.

### Coolify Cloud
Для серверов, подключенных к Coolify Cloud, для удаленного управления должен быть открыт только SSH-порт (обычно **22**).

Если вы хотите ограничить доступ по IP-адресу, у нас есть список публичных IP, используемых Coolify Cloud:

* [IPv4-адреса](https://coolify.io/ipv4.txt)
* [IPv6-адреса](https://coolify.io/ipv6.txt)

IP-адреса Coolify Cloud меняются редко, но в случае обновлений пользователи будут уведомлены по электронной почте.

### Интеграция с GitHub
GitHub использует вебхуки для связи с Coolify. Для корректной работы:
* Убедитесь, что открыты **TCP-порты 80 и 443**.
* (Опционально) Чтобы ограничить доступ к вебхукам по IP, вы можете получить актуальный список исходящих IP-адресов GitHub здесь: https://api.github.com/meta (раздел `hooks`).

Дополнительную информацию можно найти в их [документации](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/about-githubs-ip-addresses).
