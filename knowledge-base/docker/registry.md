# Реестр контейнеров (Docker Registry)

Вы можете легко пушить собранные Docker-образы в любые реестры контейнеров с помощью Coolify.

Просто заполните поля `Docker Registry` в разделе конфигурации `General` вашего сервиса.

## Конфигурация

### Образ Docker (Docker Image)

Если это поле заполнено, Coolify будет автоматически пушить ваш собранный Docker-образ в указанный реестр.

> Если поле пустое, Coolify не будет пушить собранный образ.

### Тег образа Docker (Docker Image Tag)

Если это поле заполнено, Coolify будет автоматически пушить образ с указанным вами тегом + тег с SHA коммита git.

> Если тег пустой, Coolify запушит образ только с тегом SHA коммита git.

## Учетные данные Docker (Docker Credentials)

Учетные данные Docker (начиная с v4.0.0-beta.88) используются для аутентификации в реестрах для скачивания (pull) и отправки (push) образов.

Если вы хотите авторизовать Coolify в реестре Docker:

1. **Войдите на ваш сервер**
    Подключитесь к серверу по SSH под тем же пользователем, который настроен для вашего сервера в Coolify.

2. **Авторизуйтесь в Docker Registry**
    Выполните команду `docker login`.
    
    > Вам будет предложено ввести имя пользователя и пароль/токен реестра Docker. Эти данные зависят от используемого вами реестра.

После входа в систему Coolify автоматически обнаружит ваши учетные данные и будет использовать их.

## Режим Swarm (Swarm Mode)

Если вы деплоите в кластер Swarm, убедитесь, что ваш реестр Docker доступен со всех узлов кластера (как воркеров, так и менеджеров).

## Собственный реестр (Host your own registry)

Вы можете легко развернуть свой собственный реестр, однако помните, что он будет потреблять много дискового пространства, так как по умолчанию хранит образы локально на сервере.

Больше информации о настройке других драйверов хранилища можно найти в [официальной документации](https://distribution.github.io/distribution/storage-drivers/).

В Coolify есть готовый сервис для установки реестра в один клик.

### Настройка (Setup)

Вам нужно создать пользователя и пароль для реестра.

Это можно сделать с помощью команды [htpasswd](https://httpd.apache.org/docs/current/programs/htpasswd.html):

```bash
htpasswd -nbB test test
```

Затем перейдите в меню `Storages` и в файл `/auth/registry.password` добавьте сгенерированную пару пользователь/пароль (одна строка на пользователя).

::: warning Осторожно
Не забудьте перезапустить сервис реестра.
:::
