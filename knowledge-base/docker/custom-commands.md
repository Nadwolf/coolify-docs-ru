# Кастомные команды
Для развертывания ваших ресурсов вы можете добавлять дополнительные параметры к финальной команде `docker run`, которая используется для запуска контейнера.

::: warning Осторожно
  Некоторые нативные параметры Docker не поддерживаются, так как они могут нарушить работу функций Coolify. Если вам нужны неподдерживаемые параметры, пожалуйста, [свяжитесь с нами](/get-started/support).
:::

## Поддерживаемые параметры

- `--ip`
- `--ip6`
- `--shm-size`
- `--cap-add`
- `--cap-drop`
- `--security-opt`
- `--sysctl`
- `--device`
- `--ulimit`
- `--init`
- `--privileged`
- `--gpus`
- `--entrypoint`

## Использование

Просто добавьте нужные опции в поле `Custom Docker Options` на вкладке `General` вашего ресурса.

Пример: `--cap-add SYS_ADMIN --privileged`

## Кастомный Entrypoint

Параметр `--entrypoint` позволяет переопределить стандартную точку входа (entrypoint) Docker-образа без необходимости сборки нового кастомного образа.

### Синтаксис

Coolify поддерживает три варианта синтаксиса для entrypoint:

1. **Простой entrypoint**:
   ```bash
   --entrypoint /bin/sh
   ```

2. **Команда в кавычках с аргументами**:
   ```bash
   --entrypoint "sh -c 'npm start'"
   ```

3. **Синтаксис через знак равенства**:
   ```bash
   --entrypoint=/usr/local/bin/custom-script
   ```

### Примеры использования

#### Разные типы сервисов из одного образа

Некоторые Docker-образы (например, ServerSideUp PHP) предоставляют несколько точек входа для разных типов задач:
- Воркеры (Worker processes)
- Планировщики очередей (Queue schedulers)
- Фоновые задачи (Background jobs)
- Веб-серверы

Пример запуска Laravel Horizon воркера:
```bash
--entrypoint php --entrypoint artisan --entrypoint horizon
```

#### Кастомные скрипты инициализации

Переопределение стандартного entrypoint для запуска своего скрипта:
```bash
--entrypoint "/app/custom-init.sh"
```

### Как настроить в Coolify

1. Перейдите на вкладку **General** вашего ресурса.
2. Найдите поле **Custom Docker Options**.
3. Добавьте ваш параметр entrypoint вместе с другими опциями:
   ```bash
   --entrypoint /bin/sh --cap-add SYS_ADMIN
   ```
4. Сохраните настройки и передеплойте приложение.

::: info Примечание
Опция `--entrypoint` конвертируется в формат Docker Compose во время деплоя и поддерживается как для билдпаков Dockerfile, так и для Docker Compose.
:::
