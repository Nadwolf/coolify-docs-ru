# Переменные окружения

Вы можете определять переменные окружения для своих ресурсов, и они будут доступны вашему приложению.

> Для превью-развертываний (Preview Deployments) можно задать другие переменные окружения, чтобы, например, протестировать приложение в режиме стейджинга.

## Переменные времени сборки (Build Time Variables)

Если вы хотите установить переменные окружения для процесса сборки, отметьте галочку `Build Variable` в интерфейсе управления переменными.

## Общие переменные (Shared Variables)

Существует 3 типа общих переменных:

1. На уровне команды (**Team Based**)
2. На уровне проекта (**Project Based**)
3. На уровне окружения (**Environment Based**) — например, для production, staging и т.д.

Вы можете настроить общие переменные на соответствующих страницах.

После этого вы можете использовать эти переменные в любом месте. Например, вы определили `NODE_ENV` как `production`.

### На уровне команды

Вы можете задать их на странице `Team` и использовать через <code v-pre>{{team.NODE_ENV}}</code>. Не заменяйте слово "team" на название вашей команды — используйте буквально.

### На уровне проекта

Вы можете задать их на странице `Projects` (иконка шестеренки конкретного проекта) и использовать через <code v-pre>{{project.NODE_ENV}}</code>. Не заменяйте "project" на название вашего проекта.

### На уровне окружения

Вы можете задать их на странице `Environments` (выберите `Project`, затем иконку шестеренки нужного окружения) и использовать через <code v-pre>{{environment.NODE_ENV}}</code>. Не заменяйте "environment" на название окружения.

### Использование в Docker Compose

Внутри Coolify вы можете легко настроить их, следуя инструкциям в [Базе знаний для Docker Compose](/knowledge-base/docker/compose#shared-environment-variables).

## Предопределенные переменные

Coolify заранее определяет некоторые переменные, которые вы можете использовать. Вам просто нужно добавить переменную окружения следующим образом:

```bash
# Например, вы можете использовать системную переменную так:
MY_VARIABLE=$SOURCE_COMMIT
# В приложении переменная MY_VARIABLE будет содержать хеш коммита исходного кода.
```

### Переменные приложения

#### `COOLIFY_FQDN`
Полное доменное имя (FQDN) приложения.

#### `COOLIFY_URL`
URL-адрес(а) приложения.

#### `COOLIFY_BRANCH`
Имя ветки исходного кода.

#### `COOLIFY_RESOURCE_UUID`
Уникальный идентификатор ресурса, сгенерированный Coolify.

#### `COOLIFY_CONTAINER_NAME`
Имя контейнера, сгенерированное Coolify.

#### `SOURCE_COMMIT`
Хеш коммита исходного кода.

::: tip Кэш сборки
По умолчанию `SOURCE_COMMIT` не включается в сборку Docker для сохранения кэша. Включите опцию "Include Source Commit in Build" в общих настройках приложения, если процессу сборки нужно это значение.
:::

#### `PORT`
Если не задано: устанавливается равным первому порту из списка `Port Exposes`.

#### `HOST`
Если не задано: устанавливается равным `0.0.0.0`.

### Переменные стека сервисов

#### `SERVICE_NAME_<SERVICE>`
Имя конкретного сервиса в стеке. Например, если у вас есть сервис `web`, вы можете получить его имя через `SERVICE_NAME_WEB`. Это полезно для превью-деплоев, где имена сервисов меняются.
