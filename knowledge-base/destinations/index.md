# Места назначения (Destinations)

Места назначения в Coolify — это **сетевые эндпоинты Docker**, куда развертываются ваши приложения, базы данных и сервисы. Они представляют собой изолированные сетевые среды на ваших серверах, обеспечивающие контейнерную изоляцию и сетевое взаимодействие для ваших ресурсов.

## Что такое Места назначения?

Место назначения — это, по сути, [сеть Docker](https://docs.docker.com/engine/network/), которая выступает в качестве цели для развертывания ваших ресурсов. Когда вы развертываете приложение или базу данных, они попадают в определенное место назначения (сеть Docker) на сервере, что обеспечивает сетевую изоляцию и структурирование ваших контейнерных нагрузок.

## Типы Мест назначения

Coolify разделяет места назначения на два типа в зависимости от конфигурации сервера:

### 1. Standalone Docker (Автономный Docker)

- **Назначение**: Для развертывания на одном сервере.
- **Сценарий использования**: Самая распространенная настройка для отдельных серверов.
- **Тип сети**: [Docker bridge](https://docs.docker.com/engine/network/drivers/bridge/) или кастомные сети.

### 2. Docker Swarm

- **Назначение**: Для деплоя в кластер [Docker Swarm](https://docs.docker.com/engine/swarm/).
- **Сценарий использования**: Среды с несколькими узлами (нодами).
- **Тип сети**: [Docker overlay networks](https://docs.docker.com/engine/network/drivers/overlay/).

## Основные концепции

### Сетевая изоляция

Каждое место назначения обеспечивает сетевую изоляцию между различными деплоями. Приложения, развернутые в разных местах назначения, не могут взаимодействовать друг с другом, если это не настроено явно.

### Связь с сервером

- Каждое место назначения принадлежит ровно одному серверу.
- У одного сервера может быть несколько мест назначения.
- Места назначения привязаны к демону Docker на сервере.

### Распределение ресурсов

В одном месте назначения могут находиться ресурсы разных типов:

- **Приложения** (веб-приложения, API, микросервисы).
- **Базы данных** (PostgreSQL, MySQL, Redis, MongoDB и др.).
- **Сервисы** (установка в один клик: WordPress, Ghost и т.д.).

## Преимущества

1. **Изоляция**: Разделение на сетевом уровне между разными проектами или окружениями.
2. **Организация**: Логическая группировка связанных приложений и баз данных.
3. **Безопасность**: Предотвращение несанкционированного сетевого доступа между разными деплоями.
4. **Гибкость**: Возможность развертывать одно и то же приложение в разные места назначения или на разные серверы.
5. **Масштабируемость**: Поддержка многосерверных деплоев через добавление новых мест назначения.
