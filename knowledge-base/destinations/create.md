# Создание Мест назначения (Destinations)

Это руководство поможет вам создать новые места назначения в Coolify для развертывания ваших приложений и баз данных.

## Предварительные требования

Перед созданием места назначения убедитесь, что у вас есть:

- Как минимум один сервер, подключенный к Coolify.
- Соответствующие права доступа для управления местами назначения.
- Базовое понимание того, как работают [сети Docker](https://docs.docker.com/engine/network/).

## Создание нового Места назначения

### Способ 1: Со страницы Destinations

<ZoomableImage src="/docs/images/destinations/create-destination.webp" alt="Настройка создания места назначения" />

1. Перейдите в раздел **Destinations** в основном меню.
2. Нажмите на кнопку **+ Add**.
3. Заполните данные места назначения.

### Способ 2: Со страницы управления сервером

<ZoomableImage src="/docs/images/destinations/server-create-destination.webp" alt="Настройка создания места назначения через сервер" />

1. Перейдите в раздел **Servers** и выберите нужный сервер.
2. Откройте вкладку **Destinations**.
3. Нажмите **+ Add**.
4. Заполните данные места назначения.

## Параметры конфигурации

<ZoomableImage src="/docs/images/destinations/create-destination-modal.webp" alt="Окно создания места назначения" />

### Имя Места назначения (Destination Name)

- Генерируется автоматически на основе имени сервера и ID сети.
- Можно изменить на более понятное и описательное имя.

### Имя сети (Network Name)

- Должно быть уникальным для каждого сервера.
- Автоматически генерируется уникальный идентификатор (в формате CUID2).
- Можно изменить на более понятное имя.
- **Нельзя изменить после создания.**
- Используется как реальное имя сети Docker в системе.

### Выбор сервера (Server Selection)

- Выберите один из доступных серверов вашей команды.
- Сервер должен быть онлайн и доступен.
- Это не может быть [сервер сборки (build server)](/knowledge-base/server/build-server).
- Определяет место, где будет создана сеть Docker.

### Тип Места назначения (Destination Type)

Тип места назначения **определяется автоматически** в зависимости от конфигурации выбранного сервера:

#### Standalone Docker (Автономный Docker)

- **Выбирается автоматически**, если сервер настроен как обычный Docker-хост.
- Создает стандартную сеть Docker.
- Подходит для развертывания на одном сервере.
- Поддерживает мостовые (bridge) и кастомные сети.

#### Docker Swarm

- **Выбирается автоматически**, если сервер настроен как менеджер или рабочий узел (worker) Docker Swarm.
- Создает overlay-сети для взаимодействия между несколькими узлами.
- На сервере должен быть включен режим Docker Swarm при его настройке в Coolify.
- Продвинутая функция для кластерных развертываний.

::: tip Тип определяется конфигурацией сервера
Вы не можете вручную переключаться между Standalone Docker и Docker Swarm при создании места назначения. Тип зависит от того, как сервер был настроен при добавлении в Coolify.
:::

## Автоматическое создание сети

При создании места назначения Coolify автоматически:

1. **Создает сеть Docker** на целевом сервере.
2. **Подключает прокси** (Traefik/Caddy) к этой сети.
3. **Настраивает параметры сети** для обеспечения изоляции.
4. **Включает взаимодействие между контейнерами** внутри этой сети.

## Сканирование сетей (Network Scanning)

Вы также можете просканировать существующие сети Docker на сервере и добавить их как места назначения:

<ZoomableImage src="/docs/images/destinations/scan-networks.webp" alt="Сканирование сетей" />

1. Перейдите в раздел **Servers** и выберите ваш сервер.
2. Перейдите во вкладку **Destinations**.
3. Нажмите **Scan for Destinations**.
4. Выберите найденные сети для импорта, и Coolify создаст для них соответствующие записи.

## Валидация и ошибки

Распространенные ошибки при создании:

- **Network already added to this server**: имя сети совпадает с уже существующим.

## После создания

После того как место назначения создано:

- Оно появится в списке мест назначения.
- Станет доступным для выбора при развертывании приложений и баз данных.
- Сетевая связность будет настроена автоматически.
- Оно будет готово к размещению ваших контейнеризированных ресурсов.
