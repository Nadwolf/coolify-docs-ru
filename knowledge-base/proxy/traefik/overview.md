# Прокси Traefik

[Traefik](https://traefik.io/) — это современный обратный прокси-сервер и балансировщик нагрузки с открытым исходным кодом, разработанный для обработки входящих запросов и их маршрутизации к соответствующим сервисам. Он широко используется в экосистеме контейнеров, что делает его идеальным выбором для проектов, работающих на Coolify.

По умолчанию Coolify использует Traefik в качестве прокси, что позволяет легко управлять маршрутизацией, SSL-сертификатами и многим другим без необходимости глубоких технических знаний.

## Зачем использовать Traefik?

- Динамическое управление маршрутизацией между вашими приложениями и интернетом.
- Бесшовная интеграция с оркестраторами контейнеров, такими как Docker или Kubernetes.
- Упрощенное управление SSL/TLS сертификатами, включая поддержку [Let's Encrypt](https://letsencrypt.org/).
- Продвинутые функции, такие как балансировка нагрузки и middleware для тонкой настройки.
- Встроенный дашборд для мониторинга маршрутов и конфигураций.

## Когда не стоит использовать Traefik?

- Если вам нужен полный контроль над каждым аспектом вашего обратного прокси.
- Если вы предпочитаете использовать другое решение, например [NGINX](https://nginx.org/ru/).
- Если у вас крайне сложные или специфические правила маршрутизации, которые Traefik может не поддерживать в полной мере.

## Способы использования Traefik в Coolify

Coolify автоматически настраивает Traefik как ваш прокси. Однако вы можете настраивать и расширять функциональность Traefik под свои нужды.

Ниже перечислены ключевые функции и способы использования Traefik в Coolify:

1. [Базовая аутентификация](/knowledge-base/proxy/traefik/basic-auth) -> Добавьте защиту ваших приложений логином и паролем.

2. [Собственные SSL-сертификаты](/knowledge-base/proxy/traefik/custom-ssl-certs) -> Используйте свои SSL-сертификаты вместо автоматически генерируемых.

3. [Дашборд](/knowledge-base/proxy/traefik/dashboard) -> Включите встроенную панель управления Traefik для мониторинга в реальном времени.

4. [Динамическая конфигурация](/knowledge-base/proxy/traefik/dynamic-config) -> Управляйте изменениями конфигурации на лету (правила маршрутизации, middleware).

5. [Проверки состояния (Health Checks)](/knowledge-base/health-checks) -> Настройте проверки, чтобы убедиться в стабильной работе приложений.

6. [Балансировка нагрузки](/knowledge-base/proxy/traefik/load-balancing) -> Распределяйте трафик между несколькими инстансами приложения для улучшения производительности.

7. [Перенаправления (Redirects)](/knowledge-base/proxy/traefik/redirects) -> Настройте перенаправление с HTTP на HTTPS или создайте специфические редиректы.

8. [Wildcard-сертификаты](/knowledge-base/proxy/traefik/wildcard-certs) -> Защитите все поддомены одним SSL-сертификатом.
