# Домены

Вы можете легко добавлять свои домены в Coolify или к конкретным ресурсам.

Все поля для ввода доменов умеют генерировать конфигурации прокси на основе следующих правил:

1. Нужно использовать формат FQDN (полное доменное имя): `https://coolify.io`.
2. Можно указывать несколько доменов через запятую: `https://coolify.io,https://www.coolify.io`.
3. Можно добавлять порт к домену, чтобы прокси знал, на какой внутренний порт направлять трафик: `https://coolify.io:8080,http://api.coolify.io:3000`.

## HTTPS и SSL-сертификаты

Coolify автоматически управляет SSL/TLS сертификатами для ваших приложений. Когда вы вводите домен с протоколом `https://`, вся настройка происходит автоматически в фоновом режиме.

### Как работает автоматический HTTPS

Когда вы вводите домен с протоколом `https://` (например, `https://example.com`):

1. **Автоматическая настройка прокси** — Coolify применяет необходимые настройки к вашему обратному прокси (Traefik или Caddy) для работы приложения через HTTPS.

2. **Выпуск сертификата** — Прокси автоматически инициирует процесс запроса и установки SSL-сертификатов от [Let's Encrypt](https://letsencrypt.org?utm_source=coolify.io).

3. **Автоматическое продление** — Сертификаты продлеваются автоматически до истечения срока их действия. Сертификаты Let's Encrypt действуют 90 дней, и Coolify берет на себя их бесшовное обновление.

::: success СОВЕТ
Вам не нужно делать ничего особенного для включения HTTPS. Просто используйте `https://` при вводе домена, и Coolify позаботится обо всем остальном.
:::

### Самоподписанные сертификаты (Self-Signed)

Если автоматический выпуск сертификата от [Let's Encrypt](https://letsencrypt.org?utm_source=coolify.io) не удался, прокси Coolify предоставит самоподписанный сертификат, чтобы приложение оставалось доступным. Это означает, что сайт будет открываться, но браузеры покажут предупреждение системы безопасности.

::: warning УСТРАНЕНИЕ НЕПОЛАДОК
Если вы видите предупреждение о сертификате в браузере или приложение использует самоподписанный сертификат, обратитесь к руководству [Let's Encrypt не работает](/troubleshoot/dns-and-domains/lets-encrypt-not-working) для поиска решения.
:::

## Роутинг по путям (Path-Based Routing)

Вы можете направлять трафик на разные приложения и сервисы в зависимости от пути в URL, добавляя путь к вашему домену. Это позволяет нескольким приложениям использовать один и тот же домен, будучи доступными по разным путям.

**Формат**: `https://domain.com/path` или с указанием порта: `https://domain.com:3000/path`

::: warning УКАЗАНИЕ ПОРТА
При использовании и порта, и пути, порт должен идти **после** домена, но **перед** путем.

- Правильно: `https://coolify.io:3000/api`
- Неправильно: `https://coolify.io/api:3000`
:::

### Приоритет путей

Coolify автоматически применяет правила приоритета для роутинга по путям. Более специфичные (длинные) пути имеют приоритет перед менее специфичными:

- `/api/v2/users` → наивысший приоритет
- `/api/v2` → средний приоритет
- `/api` → низкий приоритет
- `/` → самый низкий приоритет (корень/заглушка)

Это означает, что вы можете безопасно развертывать несколько приложений на одном домене, не беспокоясь о конфликтах маршрутизации.

### Требования к состоянию (Health)

Для корректной работы роутинга по путям приложение, обслуживающее конкретный путь, должно быть **запущено и здорово (healthy)**. Если приложение становится нездоровым или перестает отвечать, трафик с этого пути будет перенаправлен на приложение, обслуживающее корневой домен (`/`).

::: tip ПРИМЕР
Если у вас есть:
- Приложение А на `https://coolify.io/` (корень)
- Приложение Б на `https://coolify.io/api`

И Приложение Б становится нездоровым, запросы к `/api` будут направлены в Приложение А.
:::

Подробнее о том, как прокси Coolify интерпретируют состояние ваших ресурсов, читайте на странице [Проверки состояния (Health Checks)](/knowledge-base/health-checks).

## Привязка нескольких доменов (Catch-All)

Coolify поддерживает мультиарендность (multitenancy). При использовании [Traefik](/knowledge-base/proxy/traefik/overview) вы можете автоматически принимать трафик с нескольких доменов, отредактировав `Container Labels` вашего приложения или сервиса и определив правило [`HostRegexp`](https://doc.traefik.io/traefik/reference/routing-configuration/http/routing/rules-and-priority/#host-and-hostregexp).

::: warning Привязка всех доменов и SSL
Прокси Coolify не сможет выпускать SSL-сертификаты для catch-all доменов (общих правил). Для поддоменов конкретного домена у вас есть возможность сгенерировать [Wildcard SSL-сертификат](/knowledge-base/proxy/traefik/wildcard-certs).
:::

## Wildcard-домен (для сервера)

Вы можете установить wildcard-домен (например: `http://example.com`) для вашего сервера, чтобы легко назначать автоматически создаваемые домены всем ресурсам, подключенным к этому серверу. [Подробнее](/knowledge-base/server/introduction#wildcard-domain)

## Валидация DNS

Начиная с версии `beta.191`, Coolify проверяет DNS-записи ваших доменов с помощью DNS-сервера Cloudflare `1.1.1.1`.

Если вы хотите использовать другой DNS-сервер, перейдите на страницу `Settings > Advanced` и измените поле `Custom DNS Servers` (список серверов через запятую).
